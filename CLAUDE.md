## 异步编译验证规则（备用）

> **核心原则**：编译验证不阻塞主流程，子代理处理后返回再继续。
> **注意**：此规则暂时停用，等待用户启用编译阶段。

### 规则

1. **子代理处理**：编译、测试、构建等耗时任务交给子代理
2. **异步执行**：主线程继续处理其他功能任务，不等待编译完成
3. **迭代修复**：子代理完成→返回结果→主线程分析→再派发修复指令→直到问题解决
4. **功能优先**：小问题不阻塞，先完成功能，后续再修复

### 执行流程

```
主线程 → 派发子代理（编译/测试）
       ↓
子代理 → 执行编译验证
       ↓
返回结果 → 主线程分析
       ↓
如有错误 → 派发修复指令给子代理
       ↓
重复直到通过
```

### 子代理指令

| 用途 | 指令格式 |
|------|----------|
| 派发编译任务 | `claude -p "在 [项目路径] 执行 [编译命令]，返回结果摘要"` |
| 派发测试任务 | `claude -p "在 [项目路径] 执行 [测试命令]，返回通过/失败详情"` |
| 派发修复任务 | `claude -p "修复 [项目路径] 中以下错误：[错误信息]，返回修改内容"` |
| 并行验证 | `claude -p "[命令1]" && claaude -p "[命令2]"` |

### 禁止行为

- 禁止在主线程同步等待编译完成
- 禁止因小编译错误停止功能开发
- 禁止阻塞主流程等待用户确认

架构原则（强制）
================================================================================

1. 高频路径无锁
   - Tick接收、指标更新、策略判断全部无锁
   - 锁仅用于下单和资金更新
   - 锁外预检所有风控条件

2. 增量计算 O(1)
   - EMA、SMA、MACD 等指标必须增量计算
   - K线增量更新当前K线
   - 订单簿 O(log N) 更新

3. 三层指标体系
   - TR (True Range): 波动率突破判断
   - Pine颜色: 趋势信号 (MACD + EMA10/20 + RSI)
   - 价格位置: 周期极值判断 (close-low)/(high-low)

4. 混合持仓模式
   - 资金池 RwLock 保护（低频）
   - 策略持仓独立计算（无锁）

代码规范（强制）
================================================================================

1. 所有 lib.rs 顶部必须添加:
   #![forbid(unsafe_code)]

2. 派生宏顺序:
   #[derive(Debug, Clone, Eq, PartialEq, Serialize, Deserialize)]

3. 错误类型模式 (使用 thiserror):
   #[derive(Debug, Clone, Eq, PartialEq, Error)]
   pub enum MyError {
       #[error("描述: {0}")]
       MyVariant(String),
   }

4. 避免的问题 (来自 barter-rs 教训):
   - 禁止使用 panic!()，全部返回 Result
   - 禁止在高频路径加锁
   - 禁止过多 clone()，优先使用引用
