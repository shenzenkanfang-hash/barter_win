# H_Sandbox 沙盒测试框架

## 设计目的

**沙盒是全项目集成测试环境**，用于验证整个交易系统在真实行情数据下的完整闭环运行。

## 核心目标

### 1. 历史K线回放 → Tick注入
- 从币安API拉取历史K线数据
- 将K线转换为模拟Tick流
- 注入到系统内存供各模块消费

### 2. 完整交易闭环测试

```
┌─────────────────────────────────────────────────────────────────┐
│                     行情数据层 (b_data_source)                    │
│   历史K线 → TickGenerator → DataFeeder → 内存数据                │
└─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│                     信号生成层 (c_data_process)                  │
│   DataFeeder → K线合成器 → 指标计算(EMA/RSI/PineColor) → 交易信号 │
└─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│                     检查层 (d_checktable)                        │
│   交易信号 → 交易所规则校验 → 最小成交量/价格精度/交易对状态      │
└─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│                     风控层 (e_risk_monitor)                     │
│   规则校验 → 风控预检 → 仓位/杠杆/资金/波动率检查                 │
└─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│                     引擎层 (f_engine)                           │
│   风控通过 → 订单执行 → 订单状态追踪 → 持仓更新 → 资金更新        │
└─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│                     数据持久化                                   │
│   交易记录/持仓快照/资金流水 → SQLite → 可查询验证               │
└─────────────────────────────────────────────────────────────────┘
```

## 理想运行状态

### 阶段1: 数据准备
```
✅ 拉取历史K线 (如 BTCUSDT 2025-10-19 ~ 2025-10-21)
✅ K线分页拼接 (支持 >1000 条)
✅ 缓存到本地CSV分片
```

### 阶段2: Tick回放
```
✅ K线 → O/L/H/C 四段价格路径
✅ 每分钟60个Tick
✅ Tick注入到 DataFeeder
```

### 阶段3: 信号生成
```
✅ DataFeeder 收到 Tick
✅ K线合成器更新 1m/5m/15m/1h/1d K线
✅ 指标计算:
   - EMA5/EMA10/EMA20 趋势判断
   - RSI14 超买超卖
   - PineColor 颜色判定
✅ 信号生成:
   - 金叉/死叉
   - 趋势反转
   - 超买超卖
```

### 阶段4: 检查与风控
```
✅ CheckTable 检查:
   - symbol_rules: 最小数量/价格精度
   - position_limits: 持仓上限
   - trading_hours: 交易时间
   
✅ RiskPreChecker 检查:
   - leverage: 杠杆倍数
   - position_size: 仓位大小
   - order_value: 订单价值
   - volatility: 波动率
```

### 阶段5: 订单执行
```
✅ OrderExecutor:
   - 市价单/限价单
   - 部分成交模拟
   - 订单状态: Pending → Filled/Partial/Cancelled
   
✅ 持仓更新:
   - 开多/开空/平多/平空
   - 持仓均价计算
   - 未实现盈亏计算
```

### 阶段6: 状态验证
```
✅ 最终状态验证:
   - 账户资金: 初始资金 → 扣除手续费 → ±浮动盈亏
   - 持仓记录: 开仓记录 → 平仓记录
   - 订单记录: 完整订单生命周期
   - 风险指标: 最大回撤/盈亏比
```

## 测试场景设计

### 场景1: 趋势跟踪
```
数据: 明显趋势行情 (BTC 2025-03 上涨)
预期:
  - EMA5 > EMA20 金叉 → 做多
  - EMA5 < EMA20 死叉 → 平仓
  - 最终盈亏: 正
```

### 场景2: 震荡行情
```
数据: 横盘震荡 (BTC 2025-01)
预期:
  - RSI 超买超卖反复触发
  - 风控拦截: 订单价值过小
  - 最终盈亏: 接近0 或 负(手续费)
```

### 场景3: 极端行情
```
数据: 大幅波动 (HOTUSDT 2025-10-19)
预期:
  - 波动率超过阈值 → 禁止开仓
  - 已有持仓 → 止损检查
```

## 验证指标

| 指标 | 说明 | 验证方式 |
|------|------|----------|
| 资金一致性 | 余额 ± 浮动盈亏 = 初始资金 | 断言 |
| 持仓一致性 | 开仓量 - 平仓量 = 当前持仓 | 断言 |
| 订单完整性 | 所有订单都有 Filled/Cancelled | 日志 |
| 风控有效性 | 拦截的订单都有原因 | 日志 |
| 性能指标 | Tick处理 < 1ms | 计时 |

## 开发约束

**一、除数据层外，其他层禁止为沙盒做特别修改！**

| 层级 | 允许修改 | 说明 |
|------|---------|------|
| b_data_source | ✅ | 历史数据/模拟Tick生成（沙盒专属） |
| c_data_process | ❌ | 指标计算/信号生成（使用真实逻辑） |
| d_checktable | ❌ | 交易所规则校验（使用真实规则） |
| e_risk_monitor | ❌ | 风控检查（使用真实风控） |
| f_engine | ❌ | 订单执行/持仓管理（使用真实引擎） |

**例外情况：**
- 实际运行中本来就会遇到的问题（如网络超时处理、配置文件缺失）
- 可以修改，但必须同步到主代码库，不能是沙盒独有

```bash

cargo run -p trading-system --bin sandbox -- --symbol BTCUSDT --start 2025-10-19 --end 2025-10-21

# 带参数
cargo run -p trading-system --bin sandbox -- \
    --symbol HOTUSDT \
    --start 2025-10-19 \
    --end 2025-10-21 \
```
