# h_15m P0 级问题修复方案

**修复时间**: 2026-03-27
**方案版本**: v2.0（架构师审查后修正）
**目标**: 修复所有 P0 阻塞性问题，使模块达到可测试状态

---

## 架构师审查反馈（v1.0 → v2.0）

### 关键修正点

| 问题 | v1.0（错误） | v2.0（正确） |
|------|-------------|-------------|
| AccountProvider 接口 | `fn get_account()` 同步 | `async fn get_account()` 异步 |
| fetch_account_info 默认值 | `Decimal::ZERO`（可能通过检查） | 明确返回 `Err` 拒绝 |
| execute_once_wal 返回值 | `Result<bool>` 静默跳过 | `ExecutionResult` enum 明确区分 |

### 修正原因

1. **AccountProvider 同步接口**：在 async 上下文中调用同步接口会**阻塞 Tokio 线程池**
2. **Decimal::ZERO 默认值**：某些风控逻辑可能把 0 当作"无限制"通过检查
3. **Ok(false) 静默跳过**：难以排查问题，无日志无告警

```rust
pub async fn start(&self) {
    self.is_running.store(true, Ordering::SeqCst);
    tracing::info!(symbol = %self.config.symbol, "Trader 启动");

    // 崩溃恢复
    if let Ok(Some(record)) = self.repository.load_latest(&self.config.symbol) {
        tracing::info!(
            symbol = %self.config.symbol,
            status = ?record.trader_status,
            "已从 SQLite 恢复状态"
        );
        self.restore_from_record(&record).await;
    }

    while self.is_running.load(Ordering::SeqCst) {
        tokio::select! {
            _ = self.shutdown.notified() => {
                tracing::info!(symbol = %self.config.symbol, "收到停止信号");
                break;
            }
            _ = tokio::time::sleep(Duration::from_millis(self.config.interval_ms)) => {
                // 执行 WAL 模式交易
                if let Err(e) = self.execute_once_wal().await {
                    tracing::error!(
                        symbol = %self.config.symbol,
                        error = %e,
                        "WAL 执行失败"
                    );
                }
            }
        }
    }

    tracing::info!(symbol = %self.config.symbol, "Trader 已停止");
}
```

/// 构建待预写的记录
fn build_pending_record(&self) -> Option<TradeRecord> {
    let timestamp = chrono::Utc::now().timestamp();
    
    // 获取当前持仓快照
    let position_guard = self.position.try_read();
    let local_position = position_guard.as_ref()
        .and_then(|p| p.as_ref())
        .and_then(|p| serde_json::to_string(&*p).ok());
    
    // 获取当前状态快照
    let status_guard = self.status_machine.try_read();
    let trader_status = status_guard.as_ref()
        .and_then(|m| serde_json::to_string(&m.current_status()).ok());

    Some(TradeRecord {
        symbol: self.config.symbol.clone(),
        timestamp,
        interval_ms: self.config.interval_ms as i64,
        status: crate::h_15m::repository::RecordStatus::PENDING,
        price: self.current_price().map(|p| p.to_string()),
        volatility: self.volatility_value(),
        local_position,
        trader_status,
        // ... 其他字段保持 None
        ..Default::default()
    })
}

#### Step 2: 修改 `execute_once_wal()` 签名和逻辑

```rust
/// WAL 模式执行一次
///
/// 返回 Result<bool, ExecutorError>:
/// - Ok(true): 成功执行
/// - Ok(false): 跳过执行（非错误）
/// - Err(...): 执行失败
pub async fn execute_once_wal(&self) -> Result<bool, ExecutorError> {
    // 1. 构建记录（包含持仓快照）
    let mut record = match self.build_pending_record() {
        Some(r) => r,
        None => {
            tracing::debug!(symbol = %self.config.symbol, "无法获取持仓快照，跳过");
            return Ok(false);
        }
    };

    // 2. ID 获取带幂等处理
    let pending_id = match self.try_get_pending_id(&mut record).await {
        Ok(id) => id,
        Err(e) => {
            tracing::error!(
                symbol = %self.config.symbol,
                error = %e,
                "预写记录失败，跳过本次下单"
            );
            return Ok(false);
        }
    };

    // 3. 生成信号
    let input = match self.build_signal_input() {
        Some(i) => i,
        None => {
            self.repository.mark_failed(pending_id, "NO_SIGNAL_INPUT").ok();
            return Ok(false);
        }
    };

    let signal_output = self.signal_generator.generate(&input, &self.volatility_tier(), None);
    record.signal_json = serde_json::to_string(&signal_output).ok();

    // 4. 决策
    let (signal, order_type) = match self.decide_action_wal(&signal_output) {
        Some(s) => s,
        None => {
            self.repository.mark_failed(pending_id, "NO_SIGNAL").ok();
            return Ok(false);
        }
    };

    // 5. 获取账户和持仓信息（用于风控）
    let account_info = self.fetch_account_info().await;
    let current_side = self.current_position_side().await;
    let current_qty = self.current_position_qty().await;
    let current_price = self.current_price().unwrap_or(Decimal::ZERO);

    // 填充 WAL 记录的账户字段
    record.available_balance = account_info.available_balance.to_string();
    record.unrealized_pnl = account_info.unrealized_pnl.to_string();
    record.used_margin = account_info.used_margin.to_string();

    // 6. 执行下单
    let order_qty = self.executor.calculate_order_qty(order_type, current_qty, current_side);
    let order_value = order_qty * current_price;

    match self.executor.send_order(
        order_type,
        current_qty,
        current_side,
        order_value,
        account_info.available_balance,
        account_info.total_equity,
    ) {
        Ok(_) => {
            // 7. WAL 确认
            if let Err(e) = self.repository.confirm_record(pending_id, "OK") {
                tracing::error!(
                    symbol = %self.config.symbol,
                    id = pending_id,
                    error = %e,
                    "下单成功但确认记录失败"
                );
            }
            Ok(true)
        }
        Err(e) => {
            self.repository
                .mark_failed(pending_id, &format!("ORDER_FAILED: {}", e))
                .ok();
            Ok(false)
        }
    }
}
```

## P0-3: 风控参数使用 Decimal::MAX 绕过检查（架构修正版）

### 问题确认

**位置**: `trader.rs` 第 317-318 行

```rust
Decimal::MAX,      // available_balance: 使用最大值跳过风控
Decimal::MAX,      // total_equity: 使用最大值跳过风控
```

### 🚨 架构修正（v1.0 → v2.0）

| 问题 | v1.0（错误） | v2.0（正确） |
|------|-------------|-------------|
| AccountProvider 接口 | `fn get_account()` 同步 | `async fn get_account()` 异步 |
| fetch_account_info 默认值 | `Decimal::ZERO`（可能通过检查） | 明确返回 `Err` 拒绝 |
| execute_once_wal 返回值 | `Result<bool>` 静默跳过 | `ExecutionResult` enum 明确区分 |

### 修正原因详解

#### 问题 A: AccountProvider 同步接口会阻塞 Tokio 线程池

```rust
// ❌ v1.0 错误：同步接口在 async 中调用会阻塞
pub trait AccountProvider: Send + Sync {
    fn get_account(&self, symbol: &str) -> Result<AccountInfo, TraderError>;
}

// ✅ v2.0 正确：异步接口
pub trait AccountProvider: Send + Sync {
    async fn get_account(&self, symbol: &str) -> Result<AccountInfo, TraderError>;
}
```

#### 问题 B: Decimal::ZERO 默认值可能绕过风控

```rust
// ❌ v1.0 错误：某些风控逻辑可能把 0 当作"无限制"通过检查
async fn fetch_account_info(&self) -> AccountInfo {
    AccountInfo {
        available_balance: Decimal::ZERO,  // ❌ 危险
        // ...
    }
}

// ✅ v2.0 正确：明确拒绝
async fn fetch_account_info(&self) -> Result<AccountInfo, TraderError> {
    if let Some(ref provider) = self.account_provider {
        match provider.get_account(&self.config.symbol).await {
            Ok(info) => return Ok(info),
            Err(e) => {
                tracing::error!(symbol = %self.config.symbol, error = %e,
                    "账户服务不可用，拒绝下单");
                return Err(TraderError::AccountServiceUnavailable);
            }
        }
    }

    // 未配置账户服务时，明确拒绝
    tracing::error!(symbol = %self.config.symbol,
        "未配置 AccountProvider，无法获取风控参数，拒绝下单");
    Err(TraderError::AccountProviderNotConfigured)
}
```

#### Step 4: 修改 execute_once_wal()（ExecutionResult 版）

```rust
/// WAL 执行结果枚举
#[derive(Debug, Clone)]
pub enum ExecutionResult {
    Executed { order_id: String, order_type: OrderType },
    Skipped(&'static str),
    Failed(TraderError),
}

impl ExecutionResult {
    pub fn is_executed(&self) -> bool {
        matches!(self, ExecutionResult::Executed { .. })
    }
}

pub async fn execute_once_wal(&self) -> Result<ExecutionResult, TraderError> {
    // 1. 构建记录
    let mut record = match self.build_pending_record() {
        Some(r) => r,
        None => {
            return Ok(ExecutionResult::Skipped("无法获取持仓快照"));
        }
    };

    // 2. ID 获取带幂等处理
    let pending_id = match self.try_get_pending_id(&mut record).await {
        Ok(id) => id,
        Err(e) => {
            tracing::error!(
                symbol = %self.config.symbol,
                error = %e,
                "预写记录失败"
            );
            return Ok(ExecutionResult::Failed(TraderError::RepoError(e.to_string())));
        }
    };

    // 3. 生成信号
    let input = match self.build_signal_input() {
        Some(i) => i,
        None => {
            self.repository.mark_failed(pending_id, "NO_SIGNAL_INPUT").ok();
            return Ok(ExecutionResult::Skipped("无法构建信号输入"));
        }
    };

    let signal_output = self.signal_generator.generate(&input, &self.volatility_tier(), None);
    record.signal_json = serde_json::to_string(&signal_output).ok();

    // 4. 决策
    let (signal, order_type) = match self.decide_action_wal(&signal_output) {
        Some(s) => s,
        None => {
            self.repository.mark_failed(pending_id, "NO_SIGNAL").ok();
            return Ok(ExecutionResult::Skipped("无有效交易信号"));
        }
    };

    // 5. 获取账户信息（必须成功）
    let account_info = match self.fetch_account_info().await {
        Ok(info) => info,
        Err(e) => {
            tracing::error!(
                symbol = %self.config.symbol,
                error = %e,
                "无法获取账户信息，拒绝下单"
            );
            self.repository.mark_failed(pending_id, "ACCOUNT_INFO_FAILED").ok();
            return Ok(ExecutionResult::Failed(e));
        }
    };

    // 填充 WAL 记录
    record.available_balance = account_info.available_balance.to_string();
    record.unrealized_pnl = account_info.unrealized_pnl.to_string();

    // 6. 获取持仓状态
    let current_side = self.current_position_side().await;
    let current_qty = self.current_position_qty().await;
    let current_price = self.current_price().unwrap_or(Decimal::ZERO);

    // 7. 执行下单
    let order_qty = self.executor.calculate_order_qty(order_type, current_qty, current_side);
    let order_value = order_qty * current_price;

    match self.executor.send_order(
        order_type,
        current_qty,
        current_side,
        order_value,
        account_info.available_balance,
        account_info.total_equity,
    ) {
        Ok(result) => {
            if let Err(e) = self.repository.confirm_record(pending_id, "OK") {
                tracing::error!(
                    symbol = %self.config.symbol,
                    id = pending_id,
                    error = %e,
                    "下单成功但确认记录失败"
                );
            }
            Ok(ExecutionResult::Executed {
                order_id: result.order_id,
                order_type,
            })
        }
        Err(e) => {
            self.repository
                .mark_failed(pending_id, &format!("ORDER_FAILED: {}", e))
                .ok();
            Ok(ExecutionResult::Failed(TraderError::OrderFailed(e.to_string())))
        }
    }
}
```
