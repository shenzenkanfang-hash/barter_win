### 1.1 完整执行路径

```
Trader::start()
    │
    ├── restore_from_record()     [从 SQLite 恢复状态]
    │
    └── 主循环 loop
            │
            ├── shutdown.notified() → 退出
            │
            └── interval_ms tick
                    │
                    ├── [被注释] heartbeat()
                    ├── [被注释] execute_once_wal()  ← P0-1 核心问题
                    └── tracing::debug()  ← 实际只打日志
```

### 1.2 execute_once_wal() 执行流程

```

       └── local_position = None  ← P0-2 数据缺失
```

### 🟠 P1-1: build_signal_input() 硬编码导致信号失真

**文件**: `trader.rs`
**位置**: 第 112-127 行

```rust
fn build_signal_input(&self) -> Option<MinSignalInput> {
    let vol = self.volatility_value()?;

        tr_base_60min: dec!(0.1),
        tr_ratio_15min: Decimal::from_f64_retain(vol)?,
        zscore_14_1m: dec!(0),        // ← 硬编码0
        zscore_1h_1m: dec!(0),       // ← 硬编码0
        tr_ratio_60min_5h: dec!(0),   // ← 硬编码0
        tr_ratio_10min_1h: dec!(0),   // ← 硬编码0

        pine_bar_color: String::new(),

        price_deviation_horizontal_position: dec!(0),
```

**影响**: 7个Pin条件中至少4个永远不满足
- `zscore > 2` → 硬编码0，永远不满足
- `tr_ratio > 1` → 硬编码0，永远不满足
- `pos_norm > 80 OR < 20` → 硬编码50，永远不满足
- `acc_percentile > 90` → 硬编码0，永远不满足

**修复方案**: 从 store 和指标缓存获取真实数据

```rust
fn build_signal_input(&self) -> Option<MinSignalInput> {

        tr_base_60min: indicators.tr_base_60min,

        zscore_14_1m: indicators.zscore_14_1m,
        // ... 全部从真实数据计算
```
