## 任务一：信号输入真实数据接入（P1, 4h）

### 架构师审核修正（v1.0 → v2.0）

**修正内容**：
- ❌ 原方案：`Handle::current().block_on()` 阻塞 Tokio runtime
- ✅ 新方案：`async fn` 直接 await，避免 block_on

### 现状分析

**当前问题**:

**影响**:
- 7个Pin条件中至少4个永远不满足
- 信号输出随机错误，策略无法正常工作

### 设计方案

#### 备选方案比较

| 方案 | 优点 | 缺点 | 选择 |
|------|------|------|------|
| **A: Store 直连** | 简单直接 | 业务逻辑耦合 | ❌ |
| B: IndicatorCalculator trait | 可测试、接口清晰 | 需定义新接口 | ✅ **推荐** |
| C: 直接使用 c_data_process | 复用已有实现 | 耦合过紧 | ❌ |

#### 选择理由

**方案 B**: `IndicatorCalculator` trait 允许:
- 单元测试时 Mock 真实数据
- 未来支持多种指标计算实现
- 与 AccountProvider 模式一致

```rust
pub struct Trader {
    // ... 现有字段 ...
    
    /// 数量计算器
    /// v2.0 修正：改为 Option<> 以支持降级逻辑
    quantity_calculator: Option<MinQuantityCalculator>,
}

impl Trader {
    /// 创建带数量计算器的 Trader
    pub fn with_quantity_calculator(
        config: TraderConfig,
        executor: Arc<Executor>,
        repository: Arc<Repository>,
        store: StoreRef,
        account_provider: Option<Arc<dyn AccountProvider>>,
        indicator_calculator: Option<Arc<dyn IndicatorCalculator>>,
        qty_config: QuantityCalculatorConfig,
    ) -> Self {
        Self {
            // ... 现有初始化 ...
            quantity_calculator: Some(MinQuantityCalculator::new(MinQuantityConfig {
                base_open_qty: qty_config.base_open_qty,
                max_position_qty: qty_config.max_position_qty,
                add_multiplier: qty_config.add_multiplier,
                vol_adjustment: qty_config.vol_adjustment,
            })),
        }
    }

    /// 计算订单数量（v2.0 修正）
    /// - 使用 quantity_calculator（如果已配置）
    /// - 否则降级到 executor.calculate_order_qty()
    fn calculate_order_quantity(
        &self,
        order_type: OrderType,
        current_qty: Decimal,
        current_side: Option<PositionSide>,
        signal_output: &MinSignalOutput,
    ) -> OrderQuantity {
        let vol_tier = self.volatility_tier();
        
        // v2.0 修正：检查 quantity_calculator 而非 indicator_calculator
        match &self.quantity_calculator {
            Some(calc) => {
                // 使用 MinQuantityCalculator
                match order_type {
                    OrderType::InitialOpen => {
                        let qty = calc.calc_open_quantity(&vol_tier);
                        OrderQuantity {
                            qty,
                            full_close: false,
                            reason: format!("初始开仓 qty={}", qty),
                        }
                    }
                    OrderType::DoubleAdd => {
                        let qty = calc.calc_add_quantity(current_qty, &vol_tier);
                        OrderQuantity {
                            qty,
                            full_close: false,
                            reason: format!("加仓 qty={}", qty),
                        }
                    }
                    OrderType::DoubleClose | OrderType::DayClose => {
                        let (qty, full_close) = 
                            calc.calc_close_quantity(current_qty, signal_output);
                        OrderQuantity {
                            qty,
                            full_close,
                            reason: format!("平仓 qty={} full_close={}", qty, full_close),
                        }
                    }
                    OrderType::HedgeOpen => {
                        let qty = if current_qty > Decimal::ZERO { current_qty } else { Decimal::ZERO };
                        OrderQuantity {
                            qty,
                            full_close: false,
                            reason: format!("对冲开仓 qty={}", qty),
                        }
                    }
                    OrderType::DayHedge => {
                        let qty = current_qty.abs();
                        OrderQuantity {
                            qty,
                            full_close: false,
                            reason: format!("日线对冲 qty={}", qty),
                        }
                    }
                }
            }
            None => {
                // v2.0 修正：降级到 executor.calculate_order_qty
                let qty = self.executor.calculate_order_qty(order_type, current_qty, current_side);
                OrderQuantity {
                    qty,
                    full_close: false,
                    reason: "降级到 executor".to_string(),
                }
            }
        }
    }
}
```

## 任务三：gc_pending 定时调用（P2, 1h）

### 现状分析

**当前问题**:
```rust
// repository.rs - 已定义但从未调用
pub fn gc_pending(&self) -> Result<usize, RepoError> {
    // 清理超时 PENDING 记录
}
```

**影响**:
- 长时间运行后超时 PENDING 记录堆积
- SQLite 数据库文件膨胀
- 崩溃恢复时处理过多无效记录

### 设计方案

#### 备选方案比较

| 方案 | 优点 | 缺点 | 选择 |
|------|------|------|------|
| A: Repository 内部 spawn | 简单 | Repository 不应管理任务 | ❌ |
| B: Trader::start() 内 spawn | 位置正确 | 单点控制 | ✅ **推荐** |
| C: Engine 层统一管理 | 最灵活 | 改动过大 | ❌ |

#### 选择理由

**方案 B**: 在 `Trader::start()` 中 spawn GC 任务，职责清晰，便于控制生命周期。

### 完整实现

```rust
// ==================== Trader 新增字段和方法 ====================

use tokio::time::{interval, Duration};
use std::sync::Arc;
use parking_lot::Mutex;  // 推荐使用 parking_lot，性能更高

pub struct Trader {
    // ... 现有字段 ...
    
    /// GC 配置
    gc_config: GcConfig,
    /// GC 任务句柄（用于优雅停止）
    /// v2.0 修正：使用 Arc<Mutex<Option<...>>> 解决 &self 不可变借用问题
    gc_handle: Arc<Mutex<Option<tokio::task::JoinHandle<()>>>>,
}

impl Trader {
    /// 创建带 GC 配置的 Trader
    pub fn with_gc_config(
        config: TraderConfig,
        gc_config: GcConfig,
        // ... 其他参数 ...
    ) -> Self {
        Self {
            gc_config,
            gc_handle: Arc::new(Mutex::new(None)),
            // ... 其他初始化 ...
        }
    }

    /// 启动 GC 定时任务
    fn start_gc_task(&self) {
        let repo = Arc::clone(&self.repository);
        let timeout_secs = self.gc_config.timeout_secs;
        let interval_secs = self.gc_config.interval_secs;
        let symbol = self.config.symbol.clone();
        
        let handle = tokio::spawn(async move {
            let mut ticker = interval(Duration::from_secs(interval_secs));
            
            loop {
                ticker.tick().await;
                
                match repo.gc_pending() {
                    Ok(count) if count > 0 => {
                        tracing::info!(
                            symbol = %symbol,
                            count = count,
                            timeout_secs = timeout_secs,
                            "GC 清理完成"
                        );
                    }
                    Ok(_) => {
                        tracing::trace!(
                            symbol = %symbol,
                            "GC 检查完成，无待清理记录"
                        );
                    }
                    Err(e) => {
                        tracing::warn!(
                            symbol = %symbol,
                            error = %e,
                            "GC 清理失败"
                        );
                    }
                }
            }
        });
        
        // v2.0 修正：使用锁安全存储 handle
        let mut guard = self.gc_handle.lock();
        *guard = Some(handle);
    }

    /// 停止 GC 任务（优雅关闭）
    /// v2.0 修正：使用 Arc<Mutex<Option<...>>> 模式
    async fn stop_gc_task(&self) {
        // 获取 handle 并从 Mutex 中移除
        let handle = {
            let mut guard = self.gc_handle.lock();
            guard.take()  // 取出 handle，Mutex 变为 None
        };
        
        if let Some(h) = handle {
            handle.abort();
            match handle.await {
                Ok(_) => tracing::debug!(symbol = %self.config.symbol, "GC 任务已停止"),
                Err(e) => tracing::warn!(symbol = %self.config.symbol, error = %e, "GC 任务异常终止"),
            }
        }
    }

    /// 启动交易循环（修改版）
    pub async fn start(&self) {
        self.is_running.store(true, Ordering::SeqCst);
        tracing::info!(symbol = %self.config.symbol, "Trader 启动");

        // 启动 GC 任务
        self.start_gc_task();

        // 崩溃恢复
        if let Ok(Some(record)) = self.repository.load_latest(&self.config.symbol) {
            self.restore_from_record(&record).await;
        }

        // 主循环
        while self.is_running.load(Ordering::SeqCst) {
            tokio::select! {
                _ = self.shutdown.notified() => {
                    break;
                }
                _ = tokio::time::sleep(Duration::from_millis(self.config.interval_ms)) => {
                    self.execute_once_wal().await;
                }
            }
        }

        // 停止 GC 任务
        self.stop_gc_task().await;
        
        tracing::info!(symbol = %self.config.symbol, "Trader 已停止");
    }
}
```
