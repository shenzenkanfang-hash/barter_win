## Context

### 项目概述
- **项目名称:** Barter-Rs (Rust 量化交易系统)
- **目标交易所:** Binance Futures
- **当前架构版本:** v3.1 事件驱动协程自治架构
- **Cargo Workspace 结构:** 8 个 crate (a_common, b_data_source, b_data_mock, c_data_process, d_checktable, e_risk_monitor, f_engine, x_data)

### 核心架构设计
- **执行模型:** StrategyActor（主动驱动）+ RiskActor（被动消费）
- **通信机制:** PipelineBus（仅传递策略信号和订单事件，不含原始数据）
- **数据层:** Kline1mStream（被动接口，仅暴露 `next_message()`）
- **状态管理:** SharedStore（带版本号）+ MarketDataStore（集中式）

### 关键文件路径
| 模块 | 关键文件 |
|------|----------|
| Main 入口 | `src/main.rs` |
| 组件创建 | `src/components.rs` |
| 事件总线 | `src/event_bus.rs` (PipelineBus) |
| Actor 协程 | `src/actors.rs` (StrategyActor, RiskActor) |
| 数据存储 Trait | `crates/b_data_source/src/store/store_trait.rs` (MarketDataStore) |

## Document Structure

文档按以下 8 个章节组织：

### 1. 物理结构（Physical Structure）
- crates/ 目录布局
- 每个 crate 的职责定位
- main.rs 入口模块结构

### 2. 六层逻辑架构（Six-Layer Logical Architecture）
- Layer a_common: 通用基础（心跳、工具类）
- Layer b_data_source/b_data_mock: 数据源层
- Layer c_data_process: 数据处理层
- Layer d_checktable: 策略检查层
- Layer e_risk_monitor: 风控层
- Layer f_engine: 引擎层
- Layer x_data: 状态中心
- 层间依赖关系图

### 3. 数据生命周期（Data Lifecycle）
- Tick → Kline → Indicator → Signal → Order 的完整数据流
- PipelineStore 在数据流中的角色
- 数据不可变原则

### 4. 执行时序与并发模型（Execution Timing）
- StrategyActor 主动驱动循环
- RiskActor 被动消费模式
- PipelineBus 协调机制
- broadcast channel 用于 stop signal
- Send 约束边界说明

### 5. 接口契约（Interface Contracts）
- `MarketDataStore` trait（市场数据存储）
- `SharedStore<K, V>` trait（通用共享存储，带版本号）
- `StateCenter` trait（组件状态中心）
- `RiskService` trait（风控服务）
- `TradeLock`（交易锁，全局串行）
- `StrategyService` trait（策略服务）

### 6. 状态与存储（State and Storage）
- SharedStore 单例模式
- MarketDataStore 集中式存储
- StoreVersion 版本号追踪机制
- PipelineStore 流水线存储

### 7. 错误与边界处理（Error and Boundary Handling）
- 数据缺口处理（序列号不连续 → 告警）
- TradeLock 锁竞争（冲突 → Skip 信号）
- RiskActor 风控降级（pre_check 失败 → Cancelled）
- Kline1mStream 非 Send 类型处理（mutex guard RAII）

### 8. 设计哲学与权衡（Design Philosophy）
- 并行 vs 串行（StrategyActor 主动 vs RiskActor 被动）
- 共享存储 vs 消息传递（SharedStore vs PipelineBus）
- 数据层被动接口设计
- Send 约束与 actor 边界
