### 3.2 INT-017: 事件发布测试

#### test_eventbus_publish_tick
```rust
let event_bus = EventBus::new();
let mut tick_count = Arc::new(AtomicU64::new(0));
let counter = tick_count.clone();

event_bus.subscribe::<SimulatedTick>(move |tick| {
    counter.fetch_add(1, Ordering::Relaxed);
});

event_bus.publish(SimulatedTick {
    symbol: "BTCUSDT".to_string(),
    price: dec!(50000.0),
    qty: dec!(0.01),
    timestamp: Utc::now(),
    sequence_id: 1,
    open: dec!(50000.0),
    high: dec!(50000.0),
    low: dec!(50000.0),
    volume: dec!(1.0),
    kline_timestamp: Utc::now(),
    is_last_in_kline: false,
});

assert_eq!(tick_count.load(Ordering::Relaxed), 1);
```
**验证点**:
- [PASS] 订阅者收到发布的事件
- [PASS] 多订阅者同时收到
- [PASS] 事件顺序保持

## 4. 架构设计验证
--------------------------------------------------------------------------------
【引擎启动流程】
```
EventEngine::new(config)
    -> 初始化 EventBus
    -> 初始化 Clock
    -> 注册默认处理器
    -> state: Idle

EventEngine::start()
    -> 创建 tokio::spawn 事件循环
    -> state: Running

事件循环:
    loop {
        select! {
            event = receiver.recv() => process(event),
            _ = shutdown => break,
        }
    }
    -> state: Stopped
```

【EventBus 发布/订阅】
```
EventBus::subscribe<T>(handler)
    -> 注册到对应 Topic

EventBus::publish(event)
    -> 遍历订阅者
    -> 并行/串行调用 handler
```

【优雅关闭】
```
shutdown()
    -> 发送停止信号
    -> 等待运行中事件完成
    -> 保存引擎状态
    -> 释放资源
    -> state: Stopped
```

## 5. 集成契约
--------------------------------------------------------------------------------
【EventEngine 契约】
| 方法 | 契约 |
|------|------|
| new(config) | 返回初始化的引擎，状态为 Idle |
| start() | 返回 JoinHandle，状态变为 Running |
| shutdown() | 状态变为 Stopped，保存所有待处理订单 |
| state() | 返回当前引擎状态 |

【EventBus 契约】
| 方法 | 契约 |
|------|------|
| subscribe<T>(handler) | handler 实现 Fn(T) + Send |
| publish(event) | 所有订阅者收到相同事件 |
| unsubscribe<T>() | 移除该类型所有订阅者 |
