一、项目原生分钟级策略 (d_checktable::h_15m)
================================================================================

【MinSignalGenerator - 7条件Pin模式分钟级信号生成器】

双通道架构：
  - VolatilityTier::High  → 高速通道 (tr_base>15%, pin>=4)
  - VolatilityTier::Low/Medium → 低速通道 (日线方向参考, tr_base>10%, pin>=5)

```rust
pub fn generate(&self, input: &MinSignalInput, vol_tier: &VolatilityTier, day_direction: Option<PositionSide>) -> MinSignalOutput {
    match vol_tier {
        VolatilityTier::High => self.generate_fast_signal(input),    // 高速通道
        Low | Medium => self.generate_slow_signal(input, day_direction),  // 低速通道
    }
}
```

【Pin Condition Scoring (7 conditions, 满足>=4触发入场)】

| # | 条件 | 阈值 |
|---|------|------|
| 1 | extreme_zscore | |zscore_14_1m| > 2 OR |zscore_1h_1m| > 2 |
| 2 | extreme_vol | tr_ratio_60min_5h > 1 OR tr_ratio_10min_1h > 1 |
| 3 | extreme_pos | pos_norm_60 > 80 OR < 20 |
| 4 | extreme_speed | acc_percentile_1h > 90 |
| 5 | extreme_bg_color | pine_bg_color == "纯绿" OR "纯红" |
| 6 | extreme_bar_color | pine_bar_color == "纯绿" OR "纯红" |
| 7 | extreme_price_deviation | |price_deviation_horizontal_position| == 100 |

【信号逻辑】

| 信号 | 高速通道条件 | 低速通道条件 |
|------|-------------|-------------|
| long_entry | tr_base>15% AND price_dev<0 AND pin>=4 | 日线Long AND tr_base>10% AND pin>=5 |
| short_entry | tr_base>15% AND price_dev>0 AND pin>=4 | 日线Short AND tr_base>10% AND pin>=5 |
| long_exit | pin>=4 AND pos_norm>80 | 同左 |
| short_exit | pin>=4 AND pos_norm<20 | 同左 |

二、完整回测数据流（引擎层波动触发）
================================================================================

```
CSV 历史K线
    ↓
b_data_mock::ReplaySource::from_csv() 加载全部4320根
    ↓
逐根K线闭合时:
    ↓
    store.write_kline()  ← 关键：触发 VolatilityManager 计算
    ↓                              (滚动20根标准差)
MarketDataStoreImpl.get_volatility()  ← 获取真实波动率
    ↓
Trader.volatility_tier()               ← 基于真实波动率选择通道
    │                                  - volatility > 0.15 → High
    │                                  - volatility > 0.05 → Medium
    │                                  - volatility <= 0.05 → Low
    ↓
Trader.build_signal_input_fallback()   ← 从 Store 历史K线计算真实指标
    │                                  - zscore_14_1m (14根标准差)
    │                                  - tr_base_60min (TR百分比)
    │                                  - pos_norm_60 (价格位置)
    │                                  - tr_ratio_60min_5h
    ↓
MinSignalGenerator.generate(vol_tier)  ← 7条件Pin评分 + 通道选择
    ↓
MinSignalOutput { long_entry, short_entry, long_exit, short_exit, ... }
    ↓
Trader.decide_action()                ← 状态机决策
    ↓
d_checktable::Executor.execute()       ← 计算订单数量
    ↓
e_risk_monitor::RiskPreChecker         ← 风控预检
    ↓
b_data_mock::MockApiGateway            ← 模拟下单
    ↓
账户/持仓更新
```

【架构原则】
沙盒 = 纯数据平替，业务逻辑 0 修改：
  - 数据源替换：实盘WebSocket → 模拟CSV回放
  - 账户替换：真实交易所 → MockApiGateway
  - 信号生成：完全相同（VolatilityManager + MinSignalGenerator）
  - 风控检查：完全相同（RiskPreChecker）
  - 波动率通道：完全相同（基于 store.get_volatility() 真实值）

================================================================================
三、Trader 沙盒修复记录 (v2.4)
================================================================================

问题：原沙盒模式强制 VolatilityTier::High + 硬编码信号值
原因：设计时假设"无历史数据=无波动率"
修复：
  1. volatility_tier() 永远使用 store.get_volatility() 真实值
     - 只有 store 无数据时才降级到 Low（真正的冷启动）
  2. build_signal_input_fallback() 从 Store 历史K线计算真实指标
     - zscore, tr_base, pos_norm 等全部从历史数据计算
     - Store 无数据时返回 None（跳过本次信号）
