一、日志文件命名规范
--------------------------------------------------------------------------------

格式: [团队名]_[模块名]_[日期]_[时间戳]_[日志类型].log

示例:
- 团队A_a_common_20260328_153000_运行.log
- 团队A_a_common_20260328_153000_错误.log
- 团队A_a_common_20260328_153000_断言.log
- 团队B_b_data_mock_20260328_153500_运行.log

日志类型:
- 运行.log: 正常执行信息
- 错误.log: 错误和异常信息
- 断言.log: 测试断言结果

二、日志格式标准
--------------------------------------------------------------------------------

【时间戳格式】
ISO 8601: YYYY-MM-DDTHH:MM:SS.mmm+08:00
示例: 2026-03-28T15:30:00.123+08:00

【日志级别】
- TRACE: 最详细调试信息
- DEBUG: 调试信息
- INFO: 一般信息
- WARN: 警告
- ERROR: 错误
- FATAL: 致命错误

【日志字段】
[时间戳] [级别] [模块名] [测试点ID] [线程ID] 消息 [关键变量]

【必须记录的日志内容】

1. 测试开始
[时间戳] INFO [模块名] [测试点ID] ======================================
[时间戳] INFO [模块名] [测试点ID] 测试开始: [测试名称]
[时间戳] INFO [模块名] [测试点ID] 输入参数: [参数名]=[值], ...

2. 关键变量状态
[时间戳] DEBUG [模块名] [测试点ID] 变量状态: [变量名]=[值], [变量名]=[值]

3. 决策点记录
[时间戳] INFO [模块名] [测试点ID] 决策: [条件表达式] = [结果]
[时间戳] INFO [模块名] [测试点ID] 动作: [执行的动作描述]

4. 测试结果
[时间戳] INFO [模块名] [测试点ID] 预期结果: [描述]
[时间戳] INFO [模块名] [测试点ID] 实际结果: [描述]
[时间戳] INFO [模块名] [测试点ID] 测试结论: [通过/失败]

5. 测试结束
[时间戳] INFO [模块名] [测试点ID] 测试结束
[时间戳] INFO [模块名] [测试点ID] ======================================

三、日志保留要求
--------------------------------------------------------------------------------

1. 所有执行日志必须保留至少30天
2. 失败测试的日志必须永久保留直到问题关闭
3. 日志文件压缩后存储，节省空间
4. 汇总报告必须引用具体日志片段

四、日志分析检查清单
--------------------------------------------------------------------------------

执行测试后，检查以下日志输出是否存在:
☐ 测试开始/结束标记
☐ 输入参数完整记录
☐ 关键中间变量值
☐ 决策分支路径
☐ 预期vs实际对比
☐ 错误堆栈(如失败)
☐ 测试结论明确
