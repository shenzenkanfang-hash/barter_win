交易体系两大基石（核心设计理念）
================================================================================

本系统的交易逻辑建立在两大基石之上：

┌─────────────────────────────────────────────────────────────────────────┐
│                    基石一：插针触发器（分钟级）                           │
│                                                                         │
│   15min K线 close 移动距离 >= 13%  → 进入高速通道                       │
│   1min  价格移动距离 >= 3%      → 进入高速通道                          │
│                                                                         │
│   这是品种交易触发器 - 判断是否产生交易机会                               │
│   退出条件: TR < 1 (波动率回落)                                         │
└─────────────────────────────────────────────────────────────────────────┘
                                    +
┌─────────────────────────────────────────────────────────────────────────┐
│                    基石二：日线 Pine颜色（趋势基础）                     │
│                                                                         │
│   日线 Pine颜色 - 趋势方向和强度                                         │
│   全品种统一使用同一套日线指标                                           │
│                                                                         │
│   这是趋势判断的基础 - 决定做多还是做空                                  │
└─────────────────────────────────────────────────────────────────────────┘

市场状态与策略对应：

+--------+------------+------------------------+--------------------------------+
| 状态   | 通道       | 策略                   | 判断依据                        |
+--------+------------+------------------------+--------------------------------+
| 插针   | 高速通道   | 马丁策略               | 15min>=13% 或 1min>=3%        |
+--------+------------+------------------------+--------------------------------+
| 趋势   | 慢速通道   | 趋势策略               | 日线Pine颜色                   |
+--------+------------+------------------------+--------------------------------+
| 震荡   | 休息       | 不操作                 | 不管，忽略                     |
+--------+------------+------------------------+--------------------------------+

二、核心设计原则
================================================================================

2.1 高频路径无锁

| 操作               | 是否加锁 | 说明                     |
|--------------------|----------|------------------------|
| 行情接收/指标计算   | 否       | 高频路径，零锁开销       |
| 策略信号生成        | 否       | 读当前状态              |
| 风控预检           | 否       | 检查所有条件            |
| 订单执行           | 是       | 全局锁，串行            |
| 持仓/资金更新       | 是       | 锁内原子更新            |

2.2 增量计算 O(1)

  EMA:     O(1) 增量更新
  SMA:     O(1) 增量更新
  K线:     O(1) 更新当前K线
  订单簿:  O(log N) 更新

2.3 混合持仓模式

  资金池:   所有策略共享 RwLock 保护
  持仓结算: 每个策略独立计算，互不干扰

3.2 双通道设计

+------------+--------------------------------+--------------------------------+
| 特性       | 高速通道                        | 慢速通道                        |
+------------+--------------------------------+--------------------------------+
| 触发条件   | 15min close移动 >= 13%         | 不满足高速条件                  |
|            | 或 1min 价格移动 >= 3%          |                                |
+------------+--------------------------------+--------------------------------+
| 计算频率   | Tick级，实时响应                | 分钟级，低频                    |
+------------+--------------------------------+--------------------------------+
| 退出条件   | tr_ratio < 1 (波动率回落)      | ma5_close+PineColor平仓       |
+------------+--------------------------------+--------------------------------+
| 策略       | 马丁策略(分钟级)               | 趋势策略(日线级)                |
+------------+--------------------------------+--------------------------------+
| 指标层     | 分钟级指标(ZScore/RSI等)       | 日线级指标(Pine/TRRatio)       |
+------------+--------------------------------+--------------------------------+

通道切换逻辑（核心）：

  平时（慢速通道）：
    使用 日线级指标（全品种统一）
    - 日线 PineColor, TRRatio, ma5位置

  进入高速（插针触发）：
    切换到 分钟级指标（高速通道专用）
    - 15min>=13% 或 1min>=3%
    - ZScore, RSI 等

  退出高速：
    tr_ratio < 1（波动率回落）
    │
    └──→ 回到慢速通道 → 恢复使用日线级指标

  日线趋势平仓条件（使用最大有效周期）：
    做多平仓：ma5_close > 50 + 最大周期 Pine颜色 != 纯绿
    做空平仓：ma5_close < 50 + 最大周期 Pine颜色 != 纯红

说明:
  - 分钟级指标仅在高速通道时使用
  - 日线级指标全品种统一，慢速通道时使用
  - 震荡期间不产生任何交易信号，休息

================================================================================
四、双层指标体系
================================================================================

本系统使用双层指标体系，核心设计理念：

  日线级指标（全品种统一）：
    - 日线 PineColor, TRRatio, 价格位置
    - 平时使用（慢速通道）
    - 全品种共用同一套参数

  分钟级指标（高速通道专用）：
    - ZScore, RSI, TRRatio, PineColor 等
    - 仅在高速通道时启用
    - 快速响应，极高频

4.1 日线级指标（全品种统一）

4.1.1 PricePosition - 价格历史位置

  位置 = (close - low) / (high - low) * 100

  用途：判断价格在周期中的位置（极值区域是插针信号）

4.1.2 TRRatio - 波动率比率

  tr_ratio_5d_20d  = ATR_5d / ATR_20d
  tr_ratio_20d_60d = ATR_20d / ATR_60d

  用途：判断波动率突破程度

4.1.3 Pine颜色 - 趋势信号

基于 MACD + EMA + RSI 综合判断：

+--------+------------------+----------+----------+
| 颜色   | MACD条件         | RSI条件  | 说明     |
+--------+------------------+----------+----------+
| 纯绿   | MACD>=Signal&&MACD>=0 | -      | 强势多头 |
+--------+------------------+----------+----------+
| 淡绿   | MACD<=Signal&&MACD>=0 | -      | 弱势多头 |
+--------+------------------+----------+----------+
| 纯红   | MACD<=Signal&&MACD<=0 | -      | 强势空头 |
+--------+------------------+----------+----------+
| 淡红   | MACD>=Signal&&MACD<=0 | -      | 弱势空头 |
+--------+------------------+----------+----------+
| 紫色   | -                | RSI>=70||RSI<=30 | 极值区域 |
+--------+------------------+----------+----------+

EMA周期组（优先级从高到低）：

+--------+--------+--------+----------+
| 周期组 | 快线   | 慢线   | 应用场景  |
+--------+--------+--------+----------+
| 12_26 | EMA12  | EMA26  | 最小周期  |
+--------+--------+--------+----------+
| 20_50 | EMA20  | EMA50  | 中周期    |
+--------+--------+--------+----------+
| 100_200 | EMA100 | EMA200 | 最大周期  |
+--------+--------+--------+----------+

趋势判断规则：
  - 做多：所有有效Pine组纯绿（最小周期12_26优先）
  - 做空：所有有效Pine组紫色/纯红（最小周期12_26优先）
  - 平仓：使用最大有效周期判断

4.2 分钟级指标（高波动时启用）

4.2.1 ZScore - 价格偏离度

  zscore = (price - mean) / std

  用途：检测价格是否偏离均值（插针信号）

4.2.2 TRRatio (分钟) - 分钟级波动率

  tr_ratio_60min_5h = ATR_60min / ATR_5h
  tr_ratio_10min_1h = ATR_10min / ATR_1h

  用途：分钟级波动率判断

4.2.3 其他分钟级指标

  - pos_norm_60：60分钟价格位置
  - acc_percentile_1h：加速度百分位
  - velocity_percentile_1h：速度百分位
  - price_deviation：价格偏离度
  - pine_bg_color / pine_bar_color：分钟级Pine颜色

================================================================================
五、完整业务流（14个步骤）
================================================================================

第1步: 市场数据层 - Tick接收

  WebSocket接收Tick
      │
      ├─→ kline_1m.update(tick)      增量更新当前1m K线 O(1)
      │
      ├─→ kline_1d.update(tick)      增量更新当前1d K线 O(1)
      │
      └─→ orderbook.update(tick)      增量更新订单簿 O(log N)

第2步: 指标计算层

  K线更新触发
      │
      ├─→ ema_12_26.calculate(close)       O(1)
      ├─→ ema_20_50.calculate(close)        O(1)
      ├─→ ema_100_200.calculate(close)       O(1)
      │       │
      │       ├─→ macd.calculate(ema_fast, ema_slow)
      │       │       ├─→ signal_line.calculate(macd)
      │       │       └─→ hist = macd - signal
      │       │
      ├─→ rsi.calculate()                   O(1)
      ├─→ atr.calculate(high, low, close)   O(1)
      ├─→ tr_ratio.calculate()
      ├─→ zscore.calculate()
      └─→ pine_color.detect(macd, signal, hist, rsi)

第3步: 通道判断 - 插针触发器检测

  15min close移动距离 >= 13%
      │
      ├─→ 是 ──→ 高速通道 ──→ 马丁策略
      │
  1min 价格移动距离 >= 3%
      │
      ├─→ 是 ──→ 高速通道 ──→ 马丁策略
      │
      └─→ 否 ──→ 慢速通道 ──→ 趋势策略（日线Pine颜色）

  注意: 震荡期间不做任何操作

第4步: 高速通道 - 马丁策略（分钟级指标）

  MartinStrategy.check()
      │
      ├─→ 前置条件: TR >= 15% + 方向正确
      │
      ├─→ check_long_entry()    7条件满足>=4
      │       ├─→ zscore > 2
      │       ├─→ TR_ratio > 1
      │       ├─→ pos_norm > 80
      │       ├─→ speed > 90
      │       ├─→ pine_bg_color == 纯绿
      │       ├─→ pine_bar_color == 纯绿
      │       └─→ price_deviation == 100
      │
      ├─→ check_short_entry()   7条件满足>=4（反向）
      │
      ├─→ check_long_hedge()   6条件满足>=4（TR<15%）
      │
      └─→ check_exit_high_vol() 3条件满足>=2（退出高速）

第5步: 慢速通道 - 趋势策略（日线级指标）

  TrendStrategy.check()     日线Pine颜色作为趋势基础
      │
      ├─→ check_long_entry()    2/2条件
      │       ├─→ 所有有效Pine组纯绿（12_26优先）
      │       ├─→ TR_ratio_5d_20d > 1 或 TR_ratio_20d_60d > 1
      │       └─→ ma5_close位置 > 70
      │
      ├─→ check_short_entry()   2/2条件（反向）
      │       ├─→ 所有有效Pine组紫色/纯红
      │       ├─→ TR_ratio同上
      │       └─→ ma5_close位置 < 30
      │
      ├─→ check_long_exit()    使用最大有效周期
      │       ├─→ ma5_close > 50
      │       └─→ 最大周期非纯绿
      │
      └─→ check_short_exit()   使用最大有效周期（反向）

注意: 震荡期间 ──→ 休息，不管

第6步: 风控预检（锁外）

  RiskPreChecker.pre_check()     锁外执行
      │
      ├─→ check_account_balance()     可用资金 >= 最低保留
      ├─→ check_position_ratio()       订单金额/总权益 <= 最大比例
      ├─→ check_symbol_registered()    品种已注册到该策略
      └─→ check_volatility_mode()     波动率模式匹配

      不通过 → 返回 RejectReason
      通过   → 进入下单流程

第7步: 获取全局锁

  GlobalLock.acquire()
      失败 → 返回 LockFailed
      成功 → 进入锁内复核

第8步: 风控复核（锁内）

  RiskReChecker.re_check()      锁内再次风控
      │
      ├─→ 再次检查资金（防止并发修改）
      └─→ 实时波动率确认（市场可能已变化）

      不通过 → 释放锁，返回 RejectReason
      通过   → 执行下单

第9步: 下单执行

  OrderService.place_order()
      │
      ├─→ 市价单 → gateway.place_market_order()
      │
      └─→ 限价单 → gateway.place_limit_order()

      └─→ 等待成交回报 → OrderResult { status: FILLED / REJECTED / CANCELLED }

第10步: 释放全局锁

  GlobalLock.release()

第11步: 持仓更新

  PositionManager.update_local()
      │
      ├─→ 开多: LongQty += qty, LongAvgPrice 更新
      ├─→ 开空: ShortQty += qty, ShortAvgPrice 更新
      ├─→ 平多: LongQty -= qty, 计算已实现盈亏
      └─→ 平空: ShortQty -= qty, 计算已实现盈亏

第12步: 资金更新

  FundPool.update()
      │
      ├─→ 盈利时: CurrentBalance += pnl
      └─→ 亏损时: CurrentBalance -= pnl

第13步: 策略私有持仓更新

  StrategyPosition.update()
      └─→ 独立计算该策略的持仓和盈亏

第14步: 持久化

  PersistenceService.save()
      │
      ├─→ SQLite: 交易记录、持仓快照
      └─→ /dev/shm: K线备份、缓存

7.2 AccountPool (账户保证金池)

职责:
  1. 账户数据管理: 同步、缓存账户数据（总保证金、可用保证金等）
  2. Redis熔断保护: 当Redis连续失败时启用本地缓存兜底
  3. 数据一致性: 保证本地缓存与Redis数据的一致性

熔断机制:
  - Redis连续失败3次触发熔断
  - 熔断时有效保证金设为0，拒绝所有开仓
  - 兜底时使用最低有效保证金 MIN_EFFECTIVE_MARGIN

7.3 StrategyPool (策略保证金池)

职责:
  1. 策略保证金分配: 为不同时间级别的策略（分钟级、小时级）分配保证金
  2. 开仓计算: 根据策略配置计算可开仓名义价值
  3. 保证金占用: 管理策略实际占用的保证金和可用保证金
  4. 品种注册: 管理分钟级品种注册

7.4 OrderCheck (订单风控检查器)

职责:
  1. 订单风控检查: 检查订单是否符合风控规则
  2. 保证金预占: 使用Lua脚本原子性预占保证金，避免并发超开
  3. 订单级别限制: 单品种最大开仓限制、策略级别限制等

7.5 PnlManager 盈亏互斥机制

核心理念:
  - 低波动品种: 需要被拯救，盈亏计入总体盈亏
  - 高波动品种: 拯救者，盈利用于覆盖低波动盈亏，但自己的盈亏不计入总盈亏
  - 一次成功的高波动翻倍仓盈利可以覆盖多个低波动小仓位盈亏

================================================================================
十一、PipelineForm 全流程表单
================================================================================

核心理念: 从价格数据开始，一张表单贯穿所有层级，每层携带计算结果进入下一层。

struct PipelineForm {
    // 基础信息
    symbol:     String,           // 品种
    strategy_id: String,          // 策略ID
    period:     Period,           // 周期
    timestamp:  DateTime<Utc>,    // 时间戳
    round_id:   u64,              // 轮次ID

    // 价格数据层产出
    tick_price:     Decimal,      // 当前价格
    tick_volume:    Decimal,      // 当前成交量
    open_price:     Decimal,      // K线开盘价
    close_price:    Decimal,      // K线收盘价
    high_price:     Decimal,      // K线最高价
    low_price:      Decimal,      // K线最低价

    // 指标层产出
    ema_fast:       Decimal,      // EMA快线
    ema_slow:       Decimal,      // EMA慢线
    ema_signal:     Signal,       // EMA信号
    rsi_value:      Decimal,      // RSI值
    pine_color:     PineColor,    // Pine颜色
    price_position: Decimal,      // 价格位置 0-100

    // 策略层产出
    final_signal:   Signal,       // 最终信号
    confidence:     u8,           // 置信度 0-100
    conditions_met: u8,           // 满足条件数
    conditions_total: u8,         // 总条件数

    // 下单信息
    target_price:   Decimal,      // 目标价格
    quantity:       Decimal,      // 目标数量

    // 风控标记
    risk_flag:      bool,         // 风险标记
    reject_reason:  Option<RejectReason>,  // 拒绝原因

    // 通道信息
    is_high_freq:   bool,         // 是否高速通道
}

层级处理流程:

1. 价格数据层 (MarketDataLayer)
   输入: 原始Tick
   产出: PipelineForm { tick_price, tick_volume, open/close/high/low }

2. 指标计算层 (IndicatorLayer)
   输入: PipelineForm (价格数据填充)
   产出: PipelineForm { ema_fast, ema_slow, rsi_value, pine_color, price_position }

3. 策略判断层 (StrategyLayer)
   输入: PipelineForm (指标数据填充)
   产出: PipelineForm { final_signal, confidence, target_price, quantity }

4. Check表汇总层 (CheckTableLayer)
   输入: PipelineForm (策略数据填充)
   产出: CheckTable.entries[Key] = CheckEntry

5. 仓位决策层 (PositionDecisionLayer)
   输入: CheckTable
   产出: Vec<OrderRequest>

6. 风控预检层 (RiskPreCheckLayer)
   输入: OrderRequest
   产出: OrderRequest 或 RejectReason

7. 下单执行层 (OrderExecutionLayer)
   输入: OrderRequest (通过风控)
   产出: OrderResult
