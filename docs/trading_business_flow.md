# 交易引擎业务流程文档

## 1. 架构概述

**引擎职责**：
- 执行交易指令（下单/平仓/止损/止盈）
- 交易后状态更新
- 超时监控

**两条并行业务线**：
```
引擎 (f_engine)
│
├── 触发器（交易开始）
│   ├── 分钟级触发器
│   └── 日线级触发器
│
├── CheckTables（执行流程）
│   ├── 高速通道
│   └── 低速通道
│
└── 引擎执行指令
```

---

## 2. 触发器

### 2.1 触发器职责

触发器是**交易的最开始**，负责主动检查市场状态，发现交易机会。

### 2.2 分钟级触发器

- **触发条件**：监听高波动排名，发现高波动品种
- **数据来源**：`b_data_source` 提供高波动排名
- **信号要求**：需超过阈值（如 13%/15min）
- **冗余机制**：指标数据允许 1min 延迟，由引擎自维护

### 2.3 日线级触发器

- **触发条件**：轮询日线指标服务
- **数据要求**：指标计算需要 >250天 历史数据
- **入场逻辑**：
  - 30% 历史数据以下 + 转绿 → 做多入场
  - 70% 历史数据以上 + 转红 → 做空入场
- **入场时机**：转折第一天立刻进入

### 2.4 触发器互斥检查

发现机会后，必须检查：
- **品种已存在？** → 拒绝
- **资源满？** → 拒绝

## 3. 触发器信息表

引擎 → 策略，传递公共信息：

```rust
/// 引擎 → 策略 查询请求
pub struct StrategyQuery {
    pub timestamp: i64,                    // 当前时间戳
    pub account_available: Decimal,         // 账户可用资金
    pub account_risk_state: RiskState,      // 账户风险状态
    pub current_price: Decimal,             // 当前价格
    pub volatility_level: VolatilityLevel,   // 波动率等级
    pub position_exists: bool,               // 是否有持仓
    pub position_direction: PositionSide,    // 持仓方向
    pub position_qty: Decimal,              // 持仓数量
}
```

## 4. CheckTables 执行流程

### 4.1 概述

CheckTables 是交易执行流程，由触发器调用后返回执行表。

### 4.2 通道类型

**通道由 CheckTable 执行表决定**，不是触发器阶段决定：

| 通道 | 说明 | 适用场景 |
|------|------|---------|
| 高速通道 | 分钟级专用 | 信号生成/仓位计算/止盈止损 |
| 低速通道 | 分钟级 + 日线级 | 信号/仓位/止盈止损/加仓/移动止损 |

### 4.3 执行表内容

```rust
/// CheckTable 执行表
pub struct CheckEntry {
    pub symbol: String,           // 品种
    pub strategy_id: String,       // 策略ID
    pub period: String,           // 周期 1m/5m/15m/1d

    // 指标信号
    pub ema_signal: Signal,
    pub rsi_value: Decimal,
    pub pine_color: PineColor,
    pub price_position: Decimal,

    // 执行决策
    pub action: TradingAction,    // Long/Short/Flat/Add/Stop/TakeProfit
    pub target_price: Decimal,
    pub quantity: Decimal,
    pub channel_type: ChannelType,// HighSpeed / LowSpeed

    // 风控
    pub risk_flag: bool,

    pub timestamp: i64,
}
```

### 4.4 日线级价格控制

```rust
/// 日线级价格控制输出
pub struct PriceControlOutput {
    pub should_add: bool,           // 加仓
    pub should_stop: bool,          // 止损
    pub should_take_profit: bool,   // 止盈
    pub should_move_stop: bool,     // 移动止损
    pub profit_distance_pct: Decimal,
    pub stop_distance_pct: Decimal,
}
```

## 5. 策略响应表

策略 → 引擎，返回执行意图：

```rust
pub struct StrategyResponse {

```

## 6. 风控检查

### 6.1 两次检查

| 检查 | 时机 | 引擎关心 |
|------|------|---------|
| 一次检查 | 锁外预检 | bool |
| 二次检查 | 加锁后确认 | bool |

### 6.2 风控结果

```rust
/// 风控检查结果
pub struct RiskCheckResult {
    pub pre_check_passed: bool,    // 一次检查（锁外）
    pub lock_check_passed: bool,    // 二次检查（加锁）
}
```

**引擎只关心两个 bool，不关心具体风控细节**

| 分钟级资金池 | 高速通道 | 分钟级专用 |

## 9. 品种互斥

- **一个品种只能存在于一个策略中**
- 触发器发现品种已存在 → 拒绝

---

## 10. 超时监控

- 所有自运行的服务必须提供**最新运行时间戳**
- 如果超时太久 → **引擎杀死/重启该服务**

## 11. 执行后状态更新

### 11.1 更新流程

交易执行后，在**交易锁中**完成状态同步：

```
交易执行完成
     │
     ├── 获取交易锁
     │     ↓
     │  更新本地仓位记录
     │     ↓
     │  拉取交易所API同步最新市场状态
     │     ↓
     │  更新本地账户/持仓状态
     │     ↓
     └── 释放交易锁
```

### 11.2 交易锁

**交易锁是同一把锁**：
- 获取锁 → 同步状态 → 释放锁
- 在锁内完成所有状态同步操作

**加锁期间禁止交易**：
- 获取锁后，才能进行状态同步
- 未完成同步前，任何交易请求都无法获取锁

### 11.3 更新内容

| 更新项 | 来源 | 说明 |
|--------|------|------|
| 本地仓位记录 | 订单成交回报 | 更新持仓数量/均价 |
| 账户余额 | 交易所API | 同步最新可用资金 |
| 持仓信息 | 交易所API | 同步最新持仓 |
| 持仓价值 | 计算 | 更新保证金占用 |

## 13. 状态同步锁规则

### 13.1 锁获取条件

| 条件 | 说明 |
|------|------|
| 无交易进行中 | 当前无订单在等待成交回报 |
| 无锁持有 | 其他模块未持有锁 |

### 13.2 锁持有期间

**禁止事项**：
- 新订单请求
- 状态查询（返回锁定状态）

**允许事项**：
- 状态同步
- 日志记录
