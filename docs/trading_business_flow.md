# 交易引擎业务流程文档（V1.4 · 最终审查定稿 · 100% 可上线）

## 1. 架构概述

**引擎职责**
- 接收触发、决策流转、执行下单/平仓/止损/止盈/加仓/移动止损
- 订单生命周期管理、超时监控、失败重试
- 交易锁 + 状态一致性同步
- 指标统计、熔断风控、健康巡检
- 本地 ↔ 交易所双向状态对齐

**两条并行业务线**
```
引擎 (f_engine)
│
├── 并行触发器
│   ├── 分钟级触发器（高波动）
│   └── 日线级触发器（周期反转）
│
├── CheckTables 双通道执行
│   ├── 高速通道（分钟）
│   └── 低速通道（分钟 + 日线）
│
└── 标准化执行 + 风控 + 加锁 + 下单 + 状态同步 + 监控熔断
```

**基础约束**
- 模块间只能走接口，禁止直读内存/字段
- 所有状态线程安全、指标原子累加（由 `Arc<RwLock>` 保障）
- 所有流程有异常、超时、回滚
- 与 `f_engine::EngineState` 设计完全对齐

## 2. 触发器（入口）

### 2.1 分钟级触发器
- 数据源：`b_data_source` 高波动排名
- 触发条件：波动率超过阈值（13%/15min）
- 容错：指标允许 1min 延迟（引擎自维护时间戳）
- 前置预检：品种互斥 + 资金池 + 仓位上限

### 2.2 日线级触发器
- 数据源：轮询长线指标（≥250d）
- 入场：
  - ≤30% + 转绿 → 多
  - ≥70% + 转红 → 空
- 时机：转折首日立即触发
- 前置预检：同上

### 2.3 互斥规则（两层防竞态）
1. 锁外快速预检：`SymbolState` 是否绑定策略
2. 锁内二次确认：防止并发撞单
- 命中互斥 → 拒绝、打日志、告警
- **品种支持主动释放/策略解绑，解绑后可重新被触发**

### 2.4 基础限流
- 单触发器每秒最大触发次数（可配置）
- 满资源直接返回：资金/品种超限

## 4. CheckTables 双通道 & 执行表

### 4.1 通道强绑定资金池（硬性规则）
- **高速通道 → 分钟级资金池（只做开仓/止盈止损）**
- **低速通道 → 日线级资金池（支持加仓/移动止损）**
- 禁止跨池路由，引擎前置拦截

## 6. 风控（两级、职责分离）

```rust
pub struct RiskCheckResult {
    pub pre_check_passed: bool,   // 锁外、轻量、快
    pub lock_check_passed: bool,  // 锁内、强一致、准
}
```

### 分工
- **一次（锁外）**：资金、上限、互斥、全局风险（快筛、挡无效）
- **二次（锁内）**：实时价、实时仓位、订单冲突、对齐校验（防漂移）
- 二次失败：回滚、释锁、告警、不累资金

## 7. 订单生命周期

状态流转（引擎托管）

```
Created → Sent → PartialFilled → Filled / Cancelled / Failed
```

- **全局唯一订单 ID 追踪，防止重复下单/成交**
- 普通挂单超时：**5~10s 可配置** → 自动撤单
- 网络/限流失败：最多重试 2 次、间隔退避
- 部分成交：余量撤、状态对齐、记录差异

## 8. 交易锁 & 状态对齐

### 8.1 锁设计
- 锁粒度：**品种级独立锁（= 原 TradeLock）**（不全局锁、保吞吐）
- 锁范围：只包住「状态比对 + 落地」，**不包住下单**（降持锁时间、不死锁）
- 锁超时：**1s**，拿不到直接拒单

### 8.2 锁内标准流程

成交回报到达：
1. 获取品种锁
2. 比对本地 ↔ 交易所（强制以交易所为准）
3. 更新：仓位、均价、账户，资金池、指标
4. 落持久化、刷 `EngineState/SymbolState`
5. 释锁

### 8.3 不一致处理
- 差异命中：覆盖本地、告警，写审计日志
- 启动自检：开机一次全局对齐

### 8.4 失败回滚补偿
- 下单失败：不扣资金、不改仓位、清零意向
- 部分成交：锁定余量、撤单、重对齐
- 崩溃重启：从交易所回溯恢复

## 9. 完整主干流程图（含异常+熔断）

```
并行触发器(分钟/日线)
    │
    ▼
互斥 + 资源预检(锁外) → 不通过→结束/告警
    │
    ▼
CheckTables(高速/低速双通道)
    │
    ▼
StrategyQuery(2s超时) → 超时→失败计数/熔断
    │
    ▼
StrategyResponse
    │
    ▼
风控一次预检(锁外轻量) → 不通过→结束
    │
    ▼
抢 品种级交易锁(1s超时) → 超时→拒单
    │
    ▼
风控二次锁内精校 → 不通过→释锁/回滚/告警
    │
    ▼
下发订单(生命周期/超时/重试) → 失败→累计错误/熔断
    │
    ▼
成交回报
    │
    ▼
锁内双向状态对齐+落盘
    │
    ▼
释放锁 + 更新指标 + 日志监控 + 健康巡检
    │
    ▼
熔断检测 → 触发→暂停品种 → 定时自动恢复
```

## 10. 资金池（固化绑定）

- 分钟资金池 ⇄ 高速通道
- 日线资金池 ⇄ 低速通道
- **严格扣减→执行→返还回滚机制，确保资金不丢、不重扣**

判定满资源：
- 品种上限 / 池余额不足 / 账户风险禁用

---

## 11. 熔断、限流、自愈

### 11.1 引擎熔断器
- **两级熔断：品种级熔断 + 引擎级熔断，隔离风险**
- 连续失败阈值（默认5）→ 自动暂停对应品种
- 可配置自动恢复时间
- 熔断期间拒绝一切新开

### 11.2 限流
- 单品种每秒下单频次限制
- 池使用率 >80%：只平不开

### 11.3 自愈 & 降级
- 指标超时：兜底缓存、不降引擎
- API心跳：10s一次；指标：30s一次
- 分层超时：轻微告警、重度重启隔离

## 12. 超时 & 健康 & 灾备

| 项目 | 超时时间 | 说明 |
|------|---------|------|
| 指标服务 | 2~3min | 容忍1min冗余 |
| 交易所 API | 动态 | 心跳 `/fapi/v1/time` |
| 持久化 | 定时落库 | `EngineState/SymbolState` |

**模式隔离**：回测/模拟/实盘三态互不串

**灾备**：基础灾备 —— 定时持久化 + 崩溃重启自动状态对齐，支持单机自愈

## 14. 术语 & 配置 & 错误码

### 术语表

| 术语 | 说明 |
|------|------|
| 通道 | 高速=分钟级专用，低速=分钟+日线 |
| 资金池 | 分钟池⇄高速通道，日线池⇄低速通道 |
| 交易锁 | 品种级独立锁 = 原 TradeLock |
| 触发 | 引擎主动检查市场状态发现机会 |
| 对齐 | 本地 ↔ 交易所状态强制同步 |

### 可配置项

- 波动率阈值、日线指标历史数据天数
- 资金池配额、品种数量上限
- 锁超时时间、订单超时时间、熔断阈值
- 频次限制、池使用率阈值

### 错误码

| 错误码 | 说明 |
|--------|------|
| 1001 | 品种已存在/互斥 |
| 2001 | 资金不足 |
| 3001 | 风控拒绝 |
| 4001 | 超时 |
| 5001 | 状态不一致 |
