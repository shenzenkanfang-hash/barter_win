# 系统全景图 - 六层架构量化交易系统

## 第一层：物理结构

系统由八个 crate 物理模块构成，按依赖顺序组织：

```
a_common (工具层)
    ↓
b_data_source (数据层)
    ↓
c_data_process (信号处理层)
    ↓
d_checktable (检查层)
    ↓
e_risk_monitor (风控层)
    ↓
f_engine (引擎层)
    ↓
g_test (测试层) / h_sandbox (沙盒层)
```

每个 crate 独立编译，层与层之间通过 trait 接口通信，禁止直接访问内部数据。

**关键文件路径：**
- 工具层: `crates/a_common/src/lib.rs`
- 数据层: `crates/b_data_source/src/lib.rs`
- 信号处理层: `crates/c_data_process/src/lib.rs`
- 检查层: `crates/d_checktable/src/lib.rs`
- 风控层: `crates/e_risk_monitor/src/lib.rs`
- 引擎层: `crates/f_engine/src/lib.rs`
- 沙盒层: `crates/h_sandbox/src/lib.rs`

## 第二层：逻辑架构

六层核心职责划分：

| 层级 | 名称 | 核心职责 | 关键组件 |
|------|------|----------|----------|
| a_common | 工具层 | 通用类型定义、配置、波动率模型、Exchange 接口 | `models/`, `volatility/`, `exchange/` |
| b_data_source | 数据层 | 市场数据拉取、存储、推送 | `DataFeeder`, `MarketDataStoreImpl`, `HistoryApiClient` |
| c_data_process | 信号处理层 | 指标计算、信号生成、Pine 指标 | `SignalProcessor`, `Indicator1m`, `BigCycleCalculator` |
| d_checktable | 检查层 | 交易检查表、多条件验证、信号翻译 | `CheckTable`, `Trader`, `MinSignalGenerator` |
| e_risk_monitor | 风控层 | 风控预检、持仓管理、故障恢复 | `RiskPreChecker`, `PositionManager`, `DisasterRecovery` |
| f_engine | 引擎层 | 策略执行、Trader 管理、心跳监控 | `TraderManager`, `StrategyLoop`, `RiskChecker` trait |

**数据共享约定：**
- `b_data_source::default_store()` 返回 `Arc<MarketDataStoreImpl>` 单例
- 所有组件共享同一 Store 实例，禁止各自创建独立实例
- Store 路径: `crates/b_data_source/src/store/store_impl.rs`

## 第三层：数据流动

市场数据从进入系统到触发交易的完整旅程：

```
币安 WS/API
    ↓
b_data_source::DataFeeder::push_tick(tick: Tick)
    ↓
b_data_source::MarketDataStoreImpl::write_kline(symbol, kline, is_closed)
    ↓
┌─────────────────────────────────────────┐
│ 写入路径:                                │
│ 1. memory_store (实时分区)              │
│ 2. volatility_manager (每次写入触发计算) │
│ 3. history_store (is_closed=true 时)    │
└─────────────────────────────────────────┘
    ↓
c_data_process::SignalProcessor (信号处理器自循环)
    ↓
计算指标: EMA, RSI, PineColor, PricePosition
    ↓
d_checktable::MinSignalGenerator::generate(input, vol_tier, day_direction)
    ↓
生成交易信号: long_entry / short_entry / long_exit / short_exit
    ↓
d_checktable::Trader (状态机处理)
    ↓
e_risk_monitor::RiskPreChecker::pre_check(order, account)
    ↓
h_sandbox::ShadowBinanceGateway::execute(order_req)
    ↓
模拟成交或真实交易所
```

**关键函数签名：**

## 第四层：执行模型

系统采用并行执行模型，核心并发结构：

```
┌─────────────────────────────────────────────────────────────┐
│                    tokio::main 异步运行时                     │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────┐    ┌─────────────────┐                │
│  │ start_data_replay │    │   start_trader   │                │
│  │   (数据回放协程)   │    │   (Trader协程)   │                │
│  │                  │    │                  │                │
│  │ StreamTickGenerator │  │  Trader::start() │                │
│  │   for tick_result   │  │  loop {           │                │
│  │       ↓             │  │    execute_once() │                │
│  │  push_tick()        │  │    sleep(100ms)  │                │
│  │  write_kline()      │  │  }                │                │
│  │  sleep(150ms)       │  │                  │                │
│  └─────────────────┘    └─────────────────┘                │
│           ↓                       ↓                          │
│  ┌─────────────────────────────────────────┐                │
│  │     b_data_source::default_store()       │  ← 共享单例      │
│  │         (Arc<MarketDataStoreImpl>)       │                │
│  └─────────────────────────────────────────┘                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**并行策略：**
- `start_data_replay`: 生成 Tick 数据，睡眠 150ms/条（可配置流速）
- `start_trader`: 轮询间隔 100ms，持续检查 Store 获取新数据
- 数据注入和信号处理同时运行，通过共享 Store 同步

**时间协调：**
```rust
// sandbox_full_production.rs:248
tokio::time::sleep(tokio::time::Duration::from_millis(150)).await;

// trader.rs 配置

    pub interval_ms: u64 = 100,
    pub order_interval_ms: u64 = 100,
}
```

## 第七层：边界处理

系统如何处理缺失数据、计算失败、组件故障：

### 数据缺失处理原则
**禁止行为（红线）：**
```rust
// 禁止：默认值降级掩盖错误
let kline = store.get_current_kline().unwrap_or_default();

// 禁止：沙盒构造假指标
let fake_indicator = calculate_in_sandbox(kline);
trader.inject_signal(fake_indicator);

// 禁止：实例隔离（各组件创建独立 Store）
let data_feeder = DataFeeder::new(default_store());  // OK
let trader = Trader::new(default_store());          // OK - 使用同一单例
```

**正确做法：**
```rust
// sandbox_full_production.rs:75
let store = b_data_source::default_store().clone();  // 使用单例
let trader = Trader::new(config, executor, repository, store);

// trader.rs:可能返回 None，让调用方处理
pub fn current_price(&self) -> Option<Decimal> {
    self.store.get_current_kline(&self.config.symbol)
        .and_then(|k| k.close.parse().ok())
}
```

### Trader 错误处理
```rust
// crates/f_engine/src/core/strategy_loop.rs:80
pub async fn run(mut self) {
    loop {
        if !self.is_running() { break; }
        
        if let Err(e) = self.execute_once().await {
            self.consecutive_errors += 1;
            error!("[{}] Execute error: {}", self.symbol, e);
            
            if self.consecutive_errors >= self.config.max_consecutive_errors {
                self.stop();
                break;
            }
        } else {
            self.consecutive_errors = 0;
        }
        
        tokio::time::sleep(Duration::from_millis(self.config.interval_ms)).await;
    }
}
```

## 第八层：设计原则

指导代码实现的核心思想：

### 模块隔离原则
- **禁止** 模块之间直接访问、修改、操作对方的内部数据
- **必须** 通过公开 trait 接口通信
- 内部状态必须全部设为 `private`

### 事件驱动原则
```rust
// 正确：channel 接收 Tick
let tick_rx = gateway.subscribe().await;
while let Some(tick) = tick_rx.recv().await {
    store.write_kline(&symbol, kline, true);
}

// 废弃：轮询拉取（违反事件驱动）
let kline = data_feeder.ws_get_1m(symbol);  // 已标记 deprecated
```

### 单例共享原则
```rust
// default_store() 保证全局唯一实例
// Trader 和 DataFeeder 必须使用同一实例
let store = b_data_source::default_store().clone();
let trader = Trader::new(config, executor, repository, store);
```

### 沙盒边界原则
| 沙盒职责 | 系统职责 |
|---------|---------|
| 注入原始 Tick/K线 | 解析、存储、计算指标 |
| 拦截交易所请求 | 管理订单生命周期、状态机 |
| 模拟网络故障 | 处理超时、重试、熔断 |
| **禁止**: 预计算指标、补全数据、修改订单逻辑 |

### 波动率驱动交易
