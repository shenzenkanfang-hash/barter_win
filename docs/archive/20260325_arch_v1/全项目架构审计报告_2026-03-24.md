## 3. 跨模块依赖分析

### 3.1 实际依赖关系（从 Cargo.toml 提取）

```
f_engine ─────┬──► a_common
              ├──► b_data_source
              ├──► c_data_process
              ├──► d_checktable
              └──► e_risk_monitor

d_checktable ─┼──► a_common
              ├──► b_data_source
              ├──► c_data_process
              └──► e_risk_monitor

e_risk_monitor ──┼──► a_common
                 └──► b_data_source

c_data_process ──► b_data_source

b_data_source ───► a_common
```

### 3.2 依赖问题诊断

#### 问题 3.2.1：循环依赖风险 ⚠️

```
d_checktable 依赖 e_risk_monitor
e_risk_monitor 不依赖 d_checktable
✅ 目前无循环，但架构上存在潜在风险
```

#### 问题 3.2.2：接口层未真正生效 ❌

| 接口 | 定义位置 | 实现位置 | 问题 |
|------|----------|----------|------|
| `MarketDataProvider` | f_engine/interfaces | ❌ 未在 b_data_source 实现 | 接口存在但无实现 |
| `StrategyExecutor` | f_engine/interfaces | f_engine/strategy/executor.rs | ✅ 已实现 |
| `RiskChecker` | f_engine/interfaces | ❌ 未在 e_risk_monitor 实现 | 接口存在但无实现 |
| `ExchangeGateway` | f_engine/interfaces | f_engine/order/mock_*.rs | ⚠️ 仅 Mock |

#### 问题 3.2.3：跨层直接依赖 ⚠️

```
现状：
f_engine ───────────────► c_data_process (直接导入 TradingAction)
f_engine ───────────────► d_checktable (直接导入 PipelineForm)

理想：
f_engine ───► interfaces ───► c_data_process (通过 Trait 接口)
f_engine ───► interfaces ───► d_checktable (通过 Trait 接口)
```

## 6. 架构重构设计方案

### 6.1 重构目标

1. **统一类型系统** - 消除类型冲突，统一使用接口契约类型
2. **完善接口实现** - 为所有 interfaces/ 定义提供实现
3. **建立 Trait 依赖注入** - 核心引擎通过 Trait 接口依赖各层
4. **清晰模块边界** - 明确各 crate 职责，减少跨层直接依赖

### 6.2 重构后架构

```
┌─────────────────────────────────────────────────────────────────┐
│                         f_engine (引擎层)                         │
│                                                                  │
│  ┌──────────────────────────────────────────────────────────┐    │
│  │ interfaces/ (统一接口契约层)                               │    │
│  │                                                           │    │
│  │  MarketDataProvider ◄────── b_data_source 实现          │    │
│  │  StrategyExecutor    ◄────── c_data_process 实现         │    │
│  │  RiskChecker          ◄────── e_risk_monitor 实现          │    │
│  │  ExchangeGateway      ◄────── a_common/exchange 实现       │    │
│  │  CheckTableProvider   ◄────── d_checktable 实现           │    │
│  │                                                           │    │
│  └──────────────────────────────────────────────────────────┘    │
│                              │                                   │
│                              ▼                                   │
│  ┌──────────────────────────────────────────────────────────┐    │
│  │ core/engine_v2.rs (V1.4 核心引擎)                         │    │
│  │                                                           │    │
│  │  TradingEngine<M, S, R, G, C>                            │    │
│  │  where M: MarketDataProvider                             │    │
│  │        S: StrategyExecutor                               │    │
│  │        R: RiskChecker                                    │    │
│  │        G: ExchangeGateway                                │    │
│  │        C: CheckTableProvider                             │    │
│  │                                                           │    │
│  └──────────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────────┘
```

### 6.3 模块职责重新划分

| 层级 | Crate | 职责 | 接口 |
|------|-------|------|------|
| **L1 基础设施** | `a_common` | 类型定义、配置、日志、WS、API | `ExchangeGateway` |
| **L2 数据源** | `b_data_source` | 数据获取、K线合成、品种管理 | `MarketDataProvider` |
| **L3 信号生成** | `c_data_process` | 指标计算、信号生成 | `StrategyExecutor` |

| **L6 引擎** | `f_engine` | 协调执行、状态管理、流程控制 | 无（消费者） |

## 8. 架构优化建议

| 优化项 | 说明 | 收益 |
|--------|------|------|
| **统一错误类型** | 定义 `EngineError` 在 `a_common` | 跨层错误传播一致 |
| **提取公共类型** | 将 `f_engine::types` 移至 `a_common::types` | 类型统一 |
| **接口文档化** | 为所有 Trait 添加完整文档注释 | 可维护性提升 |
| **配置结构化** | 将分散的 `Config` 统一为 `EngineConfig` | 简化初始化 |

| 优化项 | 说明 | 工作量 |
|--------|------|--------|

| **提取 CheckTableProvider** | 在 `d_checktable` 实现 `CheckTableProvider` | 4h |
| **泛型化 engine_v2** | `TradingEngine<M, S, R, G, C>` | 8h |

### 8.3 长期优化（架构演进）
