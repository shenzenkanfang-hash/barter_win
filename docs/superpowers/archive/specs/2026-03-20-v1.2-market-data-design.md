1. 概述
================================================================================

v1.2 目标：构建高性能市场数据获取层，支持 1500+ 交易品种的 K线订阅、
订单簿深度订阅、以及滑动窗口波动率检测。

核心挑战：
- 币安 WebSocket 单连接限制 1000 streams
- 1500+ 品种无法在单连接内同时订阅 1m + 15m + 1d + Depth
- 速控限制：分片订阅 50/批，间隔 500ms

================================================================================

3. 速控限制
================================================================================

币安 WebSocket 速控规则：
- 单连接最大 1000 streams
- 分片订阅: 每批 50 streams, 间隔 500ms
- 1500 品种 × 3 类型 = 4500 streams → 需要 3+ 连接

分片订阅示例:
```json
{
  "method": "SUBSCRIBE",
  "params": ["btcusdt@kline_1m", "ethusdt@kline_1m", ...50个...],
  "id": 1709294400000
}
```
500ms 后继续下一批。

4. 波动率检测
================================================================================

VolatilityDetector 在 1m 消息内完成计算：

┌─────────────────────────────────────────────────────────────────────────────────┐
│                         VolatilityDetector                                         │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                  │
│  数据结构:                                                                         │
│  ├─ kline_1m: KLineSynthesizer (实时合成 1m K线)                               │
│  ├─ kline_15m_synth: KLineSynthesizer (内部合成 15m K线)                      │
│  ├─ kline_15m_window: Vec<KLine> (保留最近 2 根 15m K线)                      │
│  └─ thresholds: (1m: 3%, 15m: 13%)                                             │
│                                                                                  │
│  15m K线来源:                                                                     │
│  └─ 内部合成: 使用 1m K线 每15根合成1根 15m K线                                 │
│     - 收到每根 1m K线时，kline_15m_synth.update()                              │
│     - 当 15m K线完成时，更新 kline_15m_window                                   │
│                                                                                  │
│  收到 1m K线时:                                                                  │
│    ├─ 计算 1m O-C 变化率: |close - open| / open                               │
│    ├─ 内部合成 15m K线 (kline_15m_synth.update)                                │
│    └─ broadcast → SignalGenerator                                               │
│                                                                                  │
│  15m K线完成时:                                                                  │
│    ├─ 更新滑动窗口: 添加新 K线，丢弃最老的                                        │
│    └─ 计算 15m Close-Close 变化率: |curr - prev| / prev                       │
│                                                                                  │
│  高波动判断:                                                                       │
│    └─ 1m O-C > 3%  OR  15m Δ% > 13%  →  ChannelType::High                    │
│                                                                                  │
└─────────────────────────────────────────────────────────────────────────────────┘

5. 订单簿处理
================================================================================

OrderBook 结构:
```rust
pub struct OrderBook {
    symbol: String,
    bids: Vec<(Decimal, Decimal)>,  // (price, qty) 每边 20 档
    asks: Vec<(Decimal, Decimal)>,
    last_update_id: u64,
}
```

深度指标计算:
```rust
impl OrderBook {
    /// 深度指标: 买盘 / 卖盘 厚度比率
    pub fn depth_indicator(&self) -> Decimal {
        let bid_depth: Decimal = self.bids.iter().take(20).map(|(_, q)| q).sum();
        let ask_depth: Decimal = self.asks.iter().take(20).map(|(_, q)| q).sum();
        if ask_depth > dec!(0) {
            bid_depth / ask_depth
        } else {
            dec!(1)
        }
    }
}
```

6. 品种管理
================================================================================

数据来源: 每 2 分钟 GET /fapi/v1/exchangeInfo

Redis 存储结构:
- Hash Key: "exchangeInfo"
- Hash Field: <symbol> (如 "BTCUSDT")
- Hash Value: 完整规则 JSON

示例:
```
HSET exchangeInfo BTCUSDT '{"symbol":"BTCUSDT","status":"TRADING","minQty":"0.001",...}'
HGET exchangeInfo BTCUSDT
```

订阅差异同步:
1. 获取新品种列表
2. 对比当前订阅
3. 差异订阅/退订

================================================================================
6.1 历史K线存储 (Redis)
================================================================================

Redis Key 命名规则:
```
kline:1m:<symbol>   # 1分钟K线 (List, 最新320根)
kline:1d:<symbol>   # 日K线 (List, 最新100根)
kline:15m:<symbol>  # 15分钟K线 (List, 最新100根)
```

每根K线 JSON 格式:
```json
{
  "t": 1709294400000,    // 开始时间 (ms)
  "o": "50000.00",       // 开盘价
  "h": "50100.00",       // 最高价
  "l": "49900.00",       // 最低价
  "c": "50050.00",       // 收盘价
  "v": "100.5",          // 成交量
  "x": true              // 是否收盘 (IsFinal)
}
```

存储操作:
```
LPUSH kline:1m:BTCUSDT "<kline_json>"   # 新K线写入头部
LTRIM kline:1m:BTCUSDT 0 319            # 保留最新320根
LRANGE kline:1m:BTCUSDT 0 -1            # 读取全部
```

K线维护策略:
| 周期 | Redis Key | 数量 | 更新频率 | 用途 |
|------|-----------|------|----------|------|
| 1m | kline:1m:<symbol> | 320根 | 每分钟 | 实时波动率计算 |
| 15m | kline:15m:<symbol> | 100根 | 每15分钟 | 15m波动率检测 |
| 1d | kline:1d:<symbol> | 100根 | 每日 | BigCycle计算 |

注意:
- 只有 IsFinal=true 的K线才写入Redis (收盘K线)
- 实时K线 (IsFinal=false) 保留在内存 KLineSynthesizer 中

7. Depth 订阅管理
================================================================================

策略:
- 默认订阅: BTCUSDT@depth@100ms (保持连接活跃)
- 按需订阅: 高波动品种进入时，在现有连接追加订阅
- 动态取消: 品种退出高波动时，取消 Depth 订阅

实现:
```rust
pub struct DepthSubscriptionManager {
    conn: WebSocketConn,
    subscribed: FnvHashSet<String>,
    default_symbol: String,  // "BTCUSDT"
}

impl DepthSubscriptionManager {
    /// 添加订阅 (新品种进入高速通道时)
    pub async fn add(&mut self, symbol: &str) -> Result<(), MarketError> {
        if !self.subscribed.contains(symbol) {
            self.conn.send_subscribe(&format!("{}@depth@100ms", symbol)).await?;
            self.subscribed.insert(symbol.to_string());
        }
        Ok(())
    }

    /// 移除订阅 (品种退出高速通道时)
    pub async fn remove(&mut self, symbol: &str) -> Result<(), MarketError> {
        if symbol != self.default_symbol && self.subscribed.contains(symbol) {
            self.conn.send_unsubscribe(&format!("{}@depth@100ms", symbol)).await?;
            self.subscribed.remove(symbol);
        }
        Ok(())
    }
}
```

11. 订阅启动时序
================================================================================

1. DataFeeder 启动
   ├─ 连接1 (1m第一组) 建立
   ├─ 连接2 (1m第二组) 建立
   ├─ 连接3 (1d) 建立
   └─ 连接4 (Depth) 建立

2. 品种更新 (首次)
   └─ GET /fapi/v1/exchangeInfo ──→ 解析 ──→ Redis Hash

3. 分片订阅 K线
   ├─ 连接1: 分片订阅 symbol[0-750] @kline_1m (50/批, 500ms间隔)
   ├─ 连接2: 分片订阅 symbol[751-1500+] @kline_1m (50/批, 500ms间隔)
   └─ 连接3: 分片订阅所有品种 @kline_1d (50/批, 500ms间隔)

4. Depth 初始订阅
   └─ 连接4: 订阅 BTCUSDT@depth@100ms (默认)

================================================================================
12. 运行时行为
================================================================================

高波动检测流程:
1. VolatilityDetector.update(kline_1m, kline_15m)
2. 判断: 1m O-C > 3%  OR  15m Δ% > 13%
3. 如果为高波动:
   ├─ 切换 ChannelType::High
   ├─ DepthSubscriptionManager.add(symbol) ──→ 连接3 追加订阅
   └─ SignalGenerator 按 Tick 级别判断

品种订阅同步 (每2分钟):
1. GET /fapi/v1/exchangeInfo
2. 对比新旧品种列表
3. 差异订阅/退订

13. 风险与限制
================================================================================

1. 币安 WebSocket 限制可能变化，需监控
2. 1500+ 品种全量订阅需要约 30 秒完成初始化
3. Redis 连接失败时使用内存缓存，降级处理
4. 断线重连期间数据可能丢失，需外部数据源补充
