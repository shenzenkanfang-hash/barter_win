================================================================================
一、方案概述
================================================================================

本方案为量化系统离线回测解决方案，完全替代实时WebSocket网络数据源：

基于本地历史1分钟K线数据，严格复刻 Python 的 historical_retracement.py
的非固定OHLC价格路径算法，生成高仿真市场Tick数据；

模拟Tick直接写入系统原有共享内存，交易引擎保持原有逻辑主动拉取数据运行；

全程关闭WebSocket、无网络请求、无业务代码改造，无缝兼容15分钟/日线双周期策略。

三、核心算法设计（1:1 还原Python算法）
================================================================================

1. 非固定价格路径规则（核心）
--------------------------------------------------------------------------------

根据K线涨跌动态生成价格路径，不使用固定路径：

    牛市K线（收盘价 ≥ 开盘价）: 开盘 → 低点 → 高点 → 收盘
    熊市K线（收盘价 ＜ 开盘价）: 开盘 → 高点 → 低点 → 收盘


2. Tick 生成规则
--------------------------------------------------------------------------------

    按价格区间的距离比例分配每段价格的Tick数量
    每段价格使用线性插值生成连续中间价
    增加微小高斯噪声，模拟真实市场波动
    强制约束: 所有价格必须在当前K线高低点范围内
    最终价格严格对齐K线收盘价


3. 时间序列规则
--------------------------------------------------------------------------------

    1根1分钟K线 = 多档连续Tick
    时间戳均匀递增，保证回测时序绝对正确

四、全流程工作流程
================================================================================

阶段1: 历史数据加载
--------------------------------------------------------------------------------

    读取本地离线1m K线数据文件
    解析核心字段: 时间戳、开盘价、最高价、最低价、收盘价、成交量
    按时间正序排序，保证回测时序合规


阶段2: 仿真Tick生成
--------------------------------------------------------------------------------

    逐根处理K线，判断牛熊形态
    执行非固定路径算法，生成模拟Tick
    输出标准Tick数据（价格、成交量、时间戳）


阶段3: 内存数据注入
--------------------------------------------------------------------------------

    直接写入系统原有内存数据结构
    复用现有K线合成、指标计算逻辑
    无需新增存储，无架构改造


阶段4: 引擎自主运行
--------------------------------------------------------------------------------

    交易引擎按原有逻辑定时从内存拉取数据
    自动执行: 指标计算 → 双周期策略检查 → 信号生成 → 模拟执行
    引擎无感知数据源类型

12.2 核心算法1: 非固定路径选择
--------------------------------------------------------------------------------

输入: 一根1m K线 (O, H, L, C)

判断逻辑:

    if C >= O:
        → 牛市路径: O → L → H → C
    else:
        → 熊市路径: O → H → L → C

输出: 4个价格节点数组


12.3 核心算法2: Tick数量分配
--------------------------------------------------------------------------------

输入: 4个价格节点, 每根K线生成60个Tick

计算逻辑:

    1. 计算相邻节点距离: dist[0]=|O-L|, dist[1]=|L-H|, dist[2]=|H-C|
    2. 计算总距离: total = sum(dist)
    3. 按比例分配: ticks_per_seg = dist / total * 60
    4. 调整为整数，确保总和=60
    5. 最小每段2个Tick，防止过度集中


12.4 核心算法3: 中间价生成
--------------------------------------------------------------------------------

输入: 起始价, 结束价, 该段Tick数量

生成逻辑:

    1. t = linspace(0, 1, n)  // 0~1均匀向量
    2. noise = 高斯随机噪声(均值0, 标准差=区间的2%)
    3. price = 起始价 + t * (结束价-起始价) + noise
    4. 约束: price必须落在[H, L]范围内


12.5 核心算法4: 累积高低点
--------------------------------------------------------------------------------

输入: 每个Tick的价格序列

计算逻辑:

    tick_high[0] = prices[0]
    tick_high[i] = max(tick_high[i-1], prices[i])

    tick_low[0] = prices[0]
    tick_low[i] = min(tick_low[i-1], prices[i])


12.6 核心算法5: 时间戳生成
--------------------------------------------------------------------------------

输入: 父级K线timestamp, 每根K线60个Tick

计算逻辑:

    tick_timestamp[i] = parent_ts + i * 1000  // 均匀1000ms间隔
