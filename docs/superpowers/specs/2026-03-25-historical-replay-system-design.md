================================================================================
一、方案概述
================================================================================

本方案为量化系统离线回测解决方案，完全替代实时WebSocket网络数据源：

基于本地历史1分钟K线数据，严格复刻Python的historical_retracement.py的
非固定OHLC价格路径算法，生成高仿真市场Tick数据；

**流式管道模式**：模拟Tick生成后立即写入内存，引擎实时消费，无需等待全部生成；

全程关闭WebSocket、无网络请求、无业务代码改造，无缝兼容15分钟/日线双周期策略。


================================================================================
二、核心差异（V1.0 vs V1.1）
================================================================================

V1.0（旧）:
    批量生成 → 保存文件 → 再加载 → 引擎运行
    缺点: 全量生成耗时长，引擎需等待

V1.1（新）:
    流式生成 → 内存注入 → 引擎实时消费
    优点: 零延迟，引擎立即响应，边生成边回测

六、核心算法设计（1:1 还原Python算法）
================================================================================

1. 非固定价格路径规则（核心）
--------------------------------------------------------------------------------

根据K线涨跌动态生成价格路径，不使用固定路径：

    牛市K线（收盘价 ≥ 开盘价）: 开盘 → 低点 → 高点 → 收盘
    熊市K线（收盘价 ＜ 开盘价）: 开盘 → 高点 → 低点 → 收盘


2. Tick 生成规则
--------------------------------------------------------------------------------

    按价格区间的距离比例分配每段价格的Tick数量
    每段价格使用线性插值生成连续中间价
    增加微小高斯噪声，模拟真实市场波动
    强制约束: 所有价格必须在当前K线高低点范围内
    最终价格严格对齐K线收盘价


3. 时间序列规则
--------------------------------------------------------------------------------

    1根1分钟K线 = 多档连续Tick
    时间戳均匀递增，保证回测时序绝对正确

七、全流程工作流程
================================================================================

阶段1: 初始化
--------------------------------------------------------------------------------

    1. 加载历史Parquet文件（按时间正序）
    2. 初始化内存DataFeeder
    3. 启动K线合成器
    4. 启动指标计算层

--------------------------------------------------------------------------------

    for each kline in klines:
        1. 生成60个模拟Tick
        2. 写入内存 DataFeeder
        3. 触发K线合成更新
        4. 触发指标计算
        5. 驱动引擎执行一次


阶段3: 引擎自主运行
--------------------------------------------------------------------------------

    引擎按原有逻辑处理:
    指标计算 → 双周期策略检查 → 信号生成 → 模拟执行

16.2 核心算法（与Python V1.1完全一致）
--------------------------------------------------------------------------------

算法1: 非固定路径选择

    输入: K线(O, H, L, C)
    if C >= O: 路径 = [O, L, H, C]  // 牛市
    else:       路径 = [O, H, L, C]  // 熊市


算法2: Tick数量分配

    输入: 路径节点[4], 每根K线60个Tick
    1. dist[0]=|O-L|, dist[1]=|L-H|, dist[2]=|H-C|
    2. total = sum(dist)
    3. ticks_per_seg = dist / total * 60（取整）
    4. 保证每段>=2，总和=60


算法3: 中间价生成

    输入: 起始价, 结束价, 段内Tick数
    t = linspace(0, 1, n)
    noise = 高斯噪声(均值0, 标准差=区间*2%)
    price = 起始价 + t * (结束价-起始价) + noise
    clip(price, low, high)


算法4: 累积高低点

    tick_high[0] = price[0]
    tick_high[i] = max(tick_high[i-1], price[i])

    tick_low[0] = price[0]
    tick_low[i] = min(tick_low[i-1], price[i])
