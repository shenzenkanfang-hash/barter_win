#### 步骤 1.5：发送 Tick 到通道（可能背压）

**主体**: `tick_tx.send(tick).await`

**触发条件**: Tick 组装完成后立刻

**方式**: 异步发送，可能阻塞

**动作**:
```rust
// sandbox_main.rs:631
if tick_tx_clone.send(tick).await.is_err() {
    tracing::info!("通道关闭，停止注入");
    break;
}
```

**背压机制详解**:

```
通道容量: 1024

状态 1: 通道有空间 (< 1024 条)
├── send() 立即返回 Ok(())
└── 生产者继续下一轮循环

状态 2: 通道已满 (= 1024 条)
├── send() 暂停（park 当前任务）
├── Tokio 调度器切换到消费者任务
├── 消费者调用 recv().await 收到数据
├── 消费者处理完一个 Tick
├── 通道腾出空间
└── 生产者任务被唤醒，send() 完成
```

**Tokio 调度细节**:
- `send().await` 内部调用 `tokio::sync::mpsc::Sender::send()`
- 当 channel 满时，调用 `Notification::park()` 暂停当前任务
- Tokio 运行时自动调度其他任务（消费者）
- 当消费者接收数据后，调用 `Notification::notify()` 唤醒等待的生产者

**结果**: Tick 入队，send() 完成，循环继续或阻塞

**增量计算详解** (`Indicators::update()`):

```rust
// sandbox_main.rs:121-185
fn update(&mut self, tick: &Tick) {
    let price = tick.price;
    
    // 1. 添加到价格历史（滚动窗口 100）
    self.price_history.push(price);
    if self.price_history.len() > 100 {
        self.price_history.remove(0);  // O(1) 头部删除
    }
    
    // 2. 计算波动率（最近 20 个价格）
    if self.price_history.len() >= 20 {
        let n = self.price_history.len();
        let recent = &self.price_history[n - 20..];
        // 计算标准差
        let sum: Decimal = recent.iter().sum();
        let mean = sum / Decimal::from(20);
        let variance: Decimal = recent.iter()
            .map(|p| (*p - mean) * (*p - mean))
            .sum::<Decimal>() / Decimal::from(20);
        self.volatility = variance.sqrt().unwrap_or(Decimal::ZERO);
    }
    
    // 3. 计算 RSI（14 周期）
    if self.price_history.len() >= 14 {
        let mut gains = Decimal::ZERO;
        let mut losses = Decimal::ZERO;
        for window in self.price_history.windows(2) {
            let change = window[1] - window[0];
            if change > Decimal::ZERO {
                gains += change;
            } else {
                losses -= change;
            }
        }
        let avg_gain = gains / dec!(14);
        let avg_loss = losses / dec!(14);
        if avg_loss.is_zero() {
            self.rsi = Some(dec!(100));
        } else {
            let rs = avg_gain / avg_loss;
            self.rsi = Some(dec!(100) - dec!(100) / (dec!(1) + rs));
        }
    }
    
    // 4. 计算 EMA（5 和 20 周期）
    if self.price_history.len() >= 5 {
        self.ema5 = Some(Self::calc_ema(&self.price_history, 5));
    }
    if self.price_history.len() >= 20 {
        self.ema20 = Some(Self::calc_ema(&self.price_history, 20));
    }
    
    // 5. 15m 波动率（仅 K 线闭合时）
    if let Some(ref kline) = tick.kline_1m {
        if kline.is_closed {
            // 使用专门的波动率计算器
            let input = KLineInput { ... };
            let stats = self.volatility_calc.update(input);
            self.vol_15m = stats.vol_15m;
        }
    }
}
```

**时间复杂度**: O(20) 波动率 + O(14) RSI = O(1) 常数时间

**结果**: `EngineState.indicators` 更新完成

#### 步骤 2.3：策略决策

**主体**: `TradingEngine::decide()`

```rust
// sandbox_main.rs:461-489
fn decide(&self, tick: &Tick) -> Option<TradingDecision> {
    let symbol = &tick.symbol;
    let indicators = self.state.get_indicators(symbol)?;
    let position = self.state.get_position(symbol);
    
    let price = tick.price;
    let (ema5, ema20, rsi) = match (indicators.ema5, indicators.ema20, indicators.rsi) {
        (Some(e5), Some(e20), Some(r)) => (e5, e20, r),
        _ => return None,  // 数据不足不决策
    };
    
    // 无持仓 -> 检查买入条件
    if !position.has_position {
        if ema5 > ema20 && rsi < dec!(70) && rsi > dec!(30) {
            return Some(TradingDecision {
                symbol: symbol.clone(),
                action: TradingAction::Long,
                price,
                qty: dec!(0.01),
                reason: "EMA金叉".to_string(),
            });
        }
    }
    // 有持仓 -> 检查卖出条件
    else {
        if ema5 < ema20 || rsi > dec!(70) {
            return Some(TradingDecision {
                symbol: symbol.clone(),
                action: TradingAction::Flat,
                price,
                qty: dec!(0.01),
                reason: "EMA死叉或RSI超买".to_string(),
            });
        }
    }
    
    None  // 无信号
}
```

**策略逻辑**:
- 金叉买入: EMA5 > EMA20 且 RSI 在 30-70 区间
- 死叉/超买卖出: EMA5 < EMA20 或 RSI > 70
- 数据不足时直接返回 None（不决策）

**结果**: 返回 `Option<TradingDecision>`

```rust
// risk_checker.rs:75-110
fn pre_check(&self, order: &OrderRequest, account: &ExchangeAccount) -> RiskCheckResult {
    match self.mode {
        RiskMode::Strict => {
            // 1. 检查订单金额（最小 5 USDT 或 qty >= 10）
            if !self.check_order_value(order.price, order.qty) {
                return RiskCheckResult::new(false, false);
            }
            
            // 2. 检查余额是否足够
            let order_value = order.qty * order.price.unwrap_or(dec!(0));
            if order_value > account.available {
                return RiskCheckResult::new(false, false);
            }
            
            RiskCheckResult::new(true, true)
        }
        RiskMode::Bypass => RiskCheckResult::new(true, true),
        RiskMode::Audit => RiskCheckResult::new(true, true),
    }
}
```

```rust
// interceptor.rs:62-84
pub fn place_order(&self, req: EngineOrderRequest) -> Result<OrderResult, EngineError> {
    let side = match req.side {
        EngineSide::Buy => Side::Buy,
        EngineSide::Sell => Side::Sell,
    };
    
    let price = req.price.unwrap_or_else(|| {
        self.engine.read().get_current_price(&req.symbol).unwrap_or(Decimal::ZERO)

        symbol: req.symbol.clone(),
        side,
        qty: req.qty,

    Ok(result)
}
```

#### 步骤 3.2：生产者发送完毕，Drop 通道

**主体**: 生产者 async 块

**触发条件**: `StreamTickGenerator::next()` 返回 `None`（所有 K 线发完）

**方式**: 隐式 Drop

**动作**:
```rust
// sandbox_main.rs:636
tracing::info!("数据注入完成: {} ticks", tick_count);
// async 块结束，tick_tx_clone 被 Drop
```

**关键**: `Sender` 实现 `Drop` trait，当所有发送端被 Drop 时，通道进入关闭状态

---

#### 步骤 3.3：消费者收到 None，退出循环

**主体**: `TradingEngine::run()` 的 `while let` 循环

**触发条件**: 通道关闭后，`recv()` 返回 `None`

**方式**: `while let` 模式匹配失败

**动作**:
```rust
// sandbox_main.rs:399-402
while let Some(tick) = tick_rx.recv().await {
    self.on_tick(tick).await;
}
// 循环结束
tracing::info!("[Engine] {} 事件循环结束", self.symbol);
```

**结果**: 引擎事件循环退出

---

#### 步骤 3.4：tokio::join! 收尾

**主体**: Tokio 运行时

**触发条件**: 两个 async 块都完成

**动作**:
```rust
// sandbox_main.rs:580
let (tx_result, rx_result) = tokio::join! { ... };

// 两个结果都是 ()
tracing::info!("引擎退出完成");
```

**结果**: 主函数继续执行，输出结果

## 三、串行保序机制详解

### 为什么 Tick 不会乱序？

```
关键设计：单消费者 + 单生产者串行处理

生产者:
  Tick1.send() ──────────→ 通道 ──────────→ Tick2.send() ──→ ...
        ↓                              ↓
     等待背压                       等待背压
   (Tick1 入队)                  (Tick2 入队)

消费者:
  recv() → Tick1处理 → recv() → Tick2处理 → recv() → ...
              ↓                    ↓
         on_tick()完成        on_tick()完成
```

**保证**:
1. 生产者 `send().await` 是串行的，必须等前一个完成才能发下一个
2. 消费者 `recv().await` 是串行的，必须等前一个处理完才能收下一个
3. Tokio 调度确保 `send()` 和 `recv()` 交替执行
4. 通道 FIFO 语义保证顺序

## 四、背压机制详解

### 什么时候会阻塞？

```
通道容量: 1024

生产者速度 > 消费者速度 时：

时间点        生产者              通道状态        消费者
─────────────────────────────────────────────────────────
T0          发送 Tick1           [1/1024]       睡眠中
T1          发送 Tick2           [2/1024]       ...
T2          发送 Tick3           [3/1024]       唤醒，处理 Tick1
...         ...                 ...            处理 Tick2
T1000       发送 Tick1024        [1024/1024]    ...
T1001       send() 阻塞!         [满]           处理 Tick3
                                                    ...
                                                    处理 Tick1023
T1024       唤醒，send()完成     [1/1024]       收到 Tick1024
```

### 谁等谁？

```
生产者等待：
  send().await → 通道满 → park() → 等待消费者唤醒

消费者等待：
  recv().await → 通道空 → park() → 等待生产者发送

唤醒机制：
  生产者 send() 成功后 → 通知消费者 recv() 继续
  消费者 recv() 成功后 → 通知生产者 send() 继续
```

## 五、错误处理流程

### 单个 Tick 处理失败的影响范围

```
Tick N 处理失败：

情况 1：指标计算失败
├── return None（不决策）
├── 继续下一轮循环
└── 不影响 Tick N+1

情况 2：风控检查失败
├── return None（不下单）
├── 记录日志
├── 继续下一轮循环
└── 不影响 Tick N+1

情况 3：网关下单失败
├── 记录错误日志
├── self.state.stats.total_errors += 1
├── 继续下一轮循环
└── 不影响 Tick N+1
```

**关键原则**: 单个 Tick 错误不崩溃整个系统，继续处理下一个

---

## 六、关闭流程详解

### 完整关闭序列

```
1. StreamTickGenerator::next() 返回 None（所有 K 线发完）

2. 生产者 async 块结束：
   ├── tick_tx_clone 被 Drop
   ├── Sender::drop() 被调用
   └── 通道关闭，通知所有等待的接收者

3. 消费者 recv() 被唤醒：
   ├── recv() 返回 None
   └── while let 循环退出

4. TradingEngine::run() 结束：
   └── 引擎事件循环退出

5. tokio::join! 两个分支都完成：
   └── 主函数继续执行

6. 输出结果：
   ├── 账户信息
   └── 统计信息
```

### 正常退出 vs 异常退出

| 情况 | 退出方式 | 结果 |
|------|---------|------|
| 所有 Tick 处理完毕 | 生产者 Drop 通道 | 正常退出，recv() 返回 None |
| 通道已满，生产者出错 | `send().await.is_err()` | 退出注入循环，通道保持打开直到引擎退出 |
| 消费者崩溃 | Tokio 任务取消 | tokio::join! 返回错误 |
| Ctrl+C | Tokio 运行时收到信号 | 优雅关闭，所有任务取消 |

## 九、设计原则总结

### 1. 无锁设计
- 使用 `DashMap` 替代 `Arc<RwLock<HashMap>>`
- 单线程串行处理，无需锁保护

### 2. 增量计算
- 指标计算 O(1)，不复算历史
- 仅使用滚动窗口最新数据

### 3. 事件驱动
- `recv().await` 阻塞等待，无 sleep 轮询
- Tick 到达才处理，无空转

### 4. 背压自然
- 通道满时 `send().await` 阻塞
- 生产者和消费者自动同步

### 5. 错误隔离
- 单个 Tick 错误不扩散
- 继续处理下一个 Tick
