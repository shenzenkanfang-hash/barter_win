## 三、每一行的执行细节

### Phase 1: 生产者生成 Tick

```
┌─────────────────────────────────────────────────────────────────────┐
│  生产者代码位置: main() 内 async 块                                   │
│  代码行: let Some(Tick_data) = generator.next()                     │
└─────────────────────────────────────────────────────────────────────┘

主体:     StreamTickGenerator
时机:     while 循环每次迭代开始时
方式:     同步调用 generator.next()
动作:     从内部迭代器取下一条 TickData
         - TickData 包含: symbol, price, qty, timestamp
         - 60 个 TickData 组成 1 根 K 线
         - 最后一 Tick 标记 is_last_in_kline = true
```

### Phase 2: Tick 结构体构造

```
┌─────────────────────────────────────────────────────────────────────┐
│  生产者代码位置: let tick = Tick { ... }                             │
└─────────────────────────────────────────────────────────────────────┘

主体:     生产者 async 块内代码
时机:     generator.next() 返回 Some 后立即执行
方式:     同步构造 struct
动作:     将 TickData 转换为 b_data_source::Tick
         - symbol: 从 TickData 克隆
         - price: TickData.price
         - kline_1m: Some(KLine { ... })
           ├── period: Period::Minute(1)
           ├── open/high/low/close: 从 TickData 填充
           └── is_closed: ws_msg.kline.is_closed
```

### Phase 3: 发送到 Channel (关键 - 背压点)

```
┌─────────────────────────────────────────────────────────────────────┐
│  生产者代码位置: tick_tx_clone.send(tick).await                     │
└─────────────────────────────────────────────────────────────────────┘

主体:     mpsc::Sender<Tick>
时机:     Tick 结构体构造完成后立即执行
方式:     异步调用 .send(tick).await
         
         ⚠️ 这里会发生阻塞！背压机制在这里工作 ⚠️

背压机制详解:
┌─────────────────────────────────────────────────────────────────────┐
│  背压 (Backpressure) 工作原理                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. channel 容量 = 1024                                              │
│                                                                      │
│  2. 生产者连续快速发送 1025 个 Tick                                    │
│     - 第 1-1024 个: send() 立即成功，返回 Ok(())                       │
│     - 第 1025 个: send() 开始阻塞                                     │
│       └── 消费者 recv() 取走 1 个后，send() 才会返回                   │
│                                                                      │
│  3. 阻塞等待的是: 生产者任务被 park（不消耗 CPU）                      │
│     └── Tokio 调度器此时可以运行其他任务                              │
│                                                                      │
│  4. 谁在等谁:                                                         │
│     生产者 send() 阻塞 ──等待──► 消费者 recv() 处理完                 │
│     生产者 send() 继续 ──通知──► 消费者 recv() 可以继续               │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘

动作:
         - send() 尝试将 tick 写入 channel buffer
         - 如果 buffer 未满: 立即成功，await 返回
         - 如果 buffer 已满: park 当前任务，等待空间
```

### Phase 4: 消费者从 Channel 接收 (关键 - 串行点)

```
┌─────────────────────────────────────────────────────────────────────┐
│  消费者代码位置: TradingEngine::run() 内                             │
│  代码行: while let Some(tick) = tick_rx.recv().await                 │
└─────────────────────────────────────────────────────────────────────┘

主体:     mpsc::Receiver<Tick>
时机:     生产者 send() 成功写入后立即触发（Tokio 调度）
方式:     异步调用 .recv().await

串行保序详解:
┌─────────────────────────────────────────────────────────────────────┐
│  串行保序 (Serial Ordering) 实现原理                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  Tokio 的 single-threaded 语义保证:                                   │
│                                                                      │
│  1. Tokio Runtime 使用单线程调度（或配置的多线程，但这里是沙盒单核）     │
│                                                                      │
│  2. 当 recv().await 返回 Some(tick) 时:                              │
│     - Tokio 保证前一个 on_tick() 已经完全执行完毕                     │
│     - on_tick() 是同步执行的，不是 async spawn                        │
│                                                                      │
│  3. 执行流程:                                                        │
│     recv().await 返回 ──► on_tick(tick) 执行完 ──► recv().await 返回 │
│     (Tick N)              (Tick N)              (Tick N+1)            │
│                                                                      │
│  4. 不会出现的情况:                                                   │
│     ❌ recv() 返回 Tick 1                                            │
│     ❌ recv() 返回 Tick 3  (在 Tick 2 处理完之前)                     │
│     ❌ recv() 返回 Tick 2                                            │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘

动作:
         - recv() 从 channel buffer 取出一个 tick
         - 如果 buffer 为空: await 阻塞，等待 send()
         - 返回 Some(tick) 给 while let 绑定
```

### Phase 5: on_tick - 步骤 1 - 更新指标

```
┌─────────────────────────────────────────────────────────────────────┐
│  消费者代码位置: TradingEngine::on_tick() 内                          │
│  代码行: self.state.update_indicators(&tick)                          │
└─────────────────────────────────────────────────────────────────────┘

主体:     EngineState
时机:     recv() 返回 Some(tick) 后，while let 绑定完成后立即执行
方式:     同步方法调用（不是 await）

动作:     增量计算指标，O(1) 时间复杂度
┌─────────────────────────────────────────────────────────────────────┐
│  update_indicators() 内部执行:                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. DashMap.entry(symbol).or_insert_with(Indicators::default)       │
│     └── 获取或创建该 symbol 的 Indicators                             │
│                                                                      │
│  2. ind.update(tick) 增量更新:                                       │
│     ├── 添加 price 到 price_history (滚动窗口 100)                     │
│     ├── 计算波动率 volatility (基于最近 20 个价格)                      │
│     ├── 计算 RSI (14 周期)                                           │
│     ├── 计算 EMA5, EMA20                                             │
│     └── 更新 vol_15m (K 线闭合时)                                     │
│                                                                      │
│  3. 所有计算在内存中完成，无 IO，无 await                             │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────────┐

主体:     TradingEngine (self)
时机:     update_indicators() 完成后立即执行
方式:     同步方法调用

动作:     根据指标计算策略决策
┌─────────────────────────────────────────────────────────────────────┐
│  decide() 内部执行:                                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. 获取当前指标:                                                     │
│     self.state.get_indicators(symbol)                               │
│     ├── ema5: Option<Decimal>                                       │
│     ├── ema20: Option<Decimal>                                       │
│     └── rsi: Option<Decimal>                                        │
│                                                                      │
│  2. 获取当前持仓:                                                     │
│     self.state.get_position(symbol)                                  │

│                                                                      │
│  3. 策略规则:                                                        │

```
┌─────────────────────────────────────────────────────────────────────┐
│  消费者代码位置: TradingEngine::check_risk() 内                      │

动作:     调用风控预检
┌─────────────────────────────────────────────────────────────────────┐

│     Bypass 模式:                                                    │
│     ├── 直接返回 RiskCheckResult::new(true, true)                    │
│     └── 跳过所有风控检查                                             │
│                                                                      │
│  5. 返回:                                                            │
│     Some(OrderRequest) 或 None                                       │
│                                                                      │

## 四、关闭流程 - 自然退出

```
┌─────────────────────────────────────────────────────────────────────┐
│  关闭触发: 所有 Tick 发完后，生产者结束                               │
└─────────────────────────────────────────────────────────────────────┘

时间线:
───────────────────────────────────────────────────────────────────────►

│  生产者                    │  Channel              │  消费者          │
├────────────────────────────┼───────────────────────┼─────────────────┤
│                            │                       │                 │
│  generator.next() 返回 None │                       │                 │
│  ├── 退出 while 循环        │                       │                 │
│  ├── drop(tick_tx_clone)   │ tx 被 drop            │                 │
│  │   └── 发送 Dropped 事件 │                       │                 │
│  │                         │                       │ recv().await     │
│  │                         │                       │ 返回 None        │
│  │                         │                       │ ├── 退出 while   │
│  │                         │                       │ └── run() 返回    │
│                            │                       │                 │
│  tracing::info("注入完成")  │                       │ tracing::info()  │
│                            │                       │ ("引擎退出")      │
│                            │                       │                 │
├────────────────────────────┼───────────────────────┼─────────────────┤
│  生产者 async 块返回        │                       │  消费者 async 块 │
│  (tx_result = ())          │                       │  (rx_result = ())│
│                            │                       │                 │
└────────────────────────────┴───────────────────────┴─────────────────┘

自然退出原理:
┌─────────────────────────────────────────────────────────────────────┐
│  1. 生产者 generator 迭代器耗尽 → next() 返回 None                    │
│                                                                      │
│  2. 生产者 drop(tx)                                                  │
│     └── mpsc 通道发送端被 Dropped                                    │
│                                                                      │
│  3. Receiver 感知到 Sender Dropped                                   │
│     └── recv().await 返回 None (不是错误)                             │
│                                                                      │
│  4. 消费者退出 while 循环                                            │
│     └── on_tick 不会有部分执行的机会                                 │
│     └── run() 函数正常返回                                           │
│                                                                      │
│  5. tokio::join! 两个分支都完成                                     │
│     └── main() 继续执行后续代码                                       │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## 五、错误处理流程

### 5.1 单个 Tick 处理失败

```
场景: decide() 返回 None（指标未就绪）
───────────────────────────────────────────────────────────────────────►

│  Tick N                    │  预期行为                                │
├────────────────────────────┼─────────────────────────────────────────┤
│                            │                                         │
│  recv() 返回 Some(tick_N)  │                                         │
│  on_tick(tick_N) 开始      │                                         │
│  ├── update_indicators()   │ ✅ 成功，更新 DashMap                    │
│  ├── decide()              │ ❌ 返回 None (指标不足)                  │
│  │   └── return None      │                                         │
│  ├── if let Some = None   │ ⛔ 不执行后续步骤                         │
│  └── update_position()    │ ✅ 仍然执行（不影响）                     │
│                            │                                         │
│  on_tick(tick_N) 结束     │                                         │
│  recv() 返回 Some(tick_N+1))│ ✅ 继续处理下一个 Tick                   │
│                            │                                         │
└────────────────────────────┴─────────────────────────────────────────┘

结论: 单个 Tick 处理失败 → 只跳过该 Tick 的交易决策 → 继续处理下一个 Tick
```

### 5.2 风控拦截

```
场景: check_risk() 返回 None（风控拒绝）
───────────────────────────────────────────────────────────────────────►

│  决策                     │  行为                                    │
├───────────────────────────┼─────────────────────────────────────────┤
│                           │                                          │
│  decide() 返回 Some()     │                                          │
│  check_risk() 执行        │                                          │
│  ├── get_account() OK     │ ✅ 获取账户成功                           │
│  ├── pre_check() 检查     │ ❌ 返回 pre_failed()                      │
│  │   └── 余额不足         │                                          │
│  └── return None         │                                          │
│                           │                                          │
│  if let Some = None      │ ⛔ 不执行 submit_order                    │
│                           │                                          │
│  日志输出:                │                                          │
│  tracing::warn!("风控拦截")│                                          │
│                           │                                          │
└───────────────────────────┴─────────────────────────────────────────┘

结论: 风控拦截 → 只跳过该订单 → 继续处理下一个 Tick
```

### 5.3 生产者错误（不应发生）

```
场景: 生产者 async 块内部 panic
───────────────────────────────────────────────────────────────────────►

tokio::join! 行为:
┌─────────────────────────────────────────────────────────────────────┐
│  如果 join! 的任一分支 panic:                                        │
│                                                                      │
│  1. panic 会 abort 整个 tokio runtime                                │
│  2. 程序崩溃退出                                                     │
│  3. 这是我们期望的行为（不是静默失败）                                │
│                                                                      │
│  ⚠️ 但在当前实现中，生产者代码是简单的同步逻辑                         │
│     不会 panic，所以这是安全的                                         │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## 八、关键设计原则回顾

| 原则 | 实现方式 | 效果 |
|------|---------|------|
| **零轮询** | recv().await 阻塞等待 | 不浪费 CPU，不错过数据 |
| **零 sleep** | 无任何 sleep 调用 | 速率由数据源控制 |
| **串行保序** | 单 async 块内同步执行 on_tick | Tick N 一定在 Tick N+1 前处理完 |
| **背压机制** | send().await 当 buffer 满时 park | 生产者等消费者 |
| **错误不崩溃** | 单个 Tick 失败返回 None，while 继续 | 不会因为一个 Tick 导致整个系统停止 |
| **自然关闭** | Sender Dropped → recv 返回 None | 无需额外 shutdown 信号 |
| **沙盒边界** | 只生成/注入数据，不做业务逻辑 | 引擎内部才是业务边界 |
