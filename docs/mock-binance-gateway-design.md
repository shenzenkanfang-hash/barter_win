一、设计目标
================================================================================

本地维护一个 MockBinanceGateway，风控逻辑与币安完全一致，用于：

1. **模拟交易** - 不使用真实账户进行回测
2. **CSV 记录** - 生成交易记录供用户对账
3. **指标准确性验证** - 同时输出 Rust vs Python 指标对比
4. **风控逻辑验证** - 确认风控与币安一致

三、风控逻辑（与币安一致）
================================================================================

3.1 下单前风控检查顺序

OrderService.place_order()
    │
    ├─→ 1. check_account_balance()     可用余额 >= 订单金额
    │
    ├─→ 2. check_position_limit()       持仓占比 <= 限制
    │
    ├─→ 3. check_symbol_status()        品种状态正常（可交易）
    │
    ├─→ 4. check_price_deviation()      价格偏离度 <= 限制
    │
    ├─→ 5. check_margin_sufficient()     保证金充足
    │
    └─→ 6. check_order_frequency()      订单频率 <= 限制

    └─→ 通过 → 立即成交（Market Order）
    └─→ 不通过 → 返回 RejectReason

3.2 持仓风控检查（实时）

PositionMonitor.check()
    │
    ├─→ 计算当前持仓的保证金率
    │       margin_ratio = maintenance_margin / (qty * price)
    │
    ├─→ 检查维持保证金是否足够
    │       if margin_ratio < 0.005 (0.5%) → 触发强平
    │
    └─→ 触发强平时
            ├─→ 平掉全部持仓
            ├─→ 扣除占用保证金
            └─→ 记录强平日志

3.3 保证金计算规则

开仓保证金：
  required_margin = qty * price / leverage

例如：
  - 买入 0.1 BTC @ 70000，10x杠杆
  - required_margin = 0.1 * 70000 / 10 = 700 USDT

维持保证金：
  maintenance_margin = qty * avg_price * maintenance_rate
  maintenance_rate = 0.5%（币安USDT永续合约）

例如（续）：
  - 持仓 0.1 BTC @ 70000
  - maintenance_margin = 0.1 * 70000 * 0.005 = 35 USDT

强制平仓条件：
  当前保证金 < 维持保证金
  即：margin_ratio < 0.5%

3.4 风控配置

+------------------------+------------------+
| 配置项                 | 值               |
+------------------------+------------------+
| 最大持仓比例           | 95%              |
| 最低保留比例           | 5%               |
| 最大订单金额           | 总权益 * 10%      |
| 维持保证金率           | 0.5%             |
| 订单频率限制           | 10次/秒          |
| 价格偏离限制           | 涨跌停限制        |
+------------------------+------------------+

================================================================================
四、成交机制
================================================================================

4.1 市价单立即成交

Market Order 处理：
    │
    ├─→ 检查风控（账户余额、保证金、持仓限制）
    │
    ├─→ 通过
    │       ├─→ 计算成交价格（以当前买一/卖一价格）
    │       ├─→ 更新持仓（数量、价格）
    │       ├─→ 计算/扣除保证金
    │       ├─→ 计算未实现盈亏
    │       └─→ 生成成交记录
    │
    └─→ 不通过 → 订单拒绝

4.2 限价单（简化处理）

Limit Order 处理：
    │
    ├─→ 检查风控
    │
    ├─→ 挂单到订单簿
    │
    └─→ 未来扩展：撮合引擎匹配成交

注意：v1.1 简化处理，限价单暂不实现撮合，仅记录订单。

- Taker 手续费：0.04%（吃单）

  commission = qty * price * 0.0004

八、测试场景
================================================================================

8.1 正常交易流程

1. 账户有足够余额
2. 下单 → 成交 → 持仓更新 → 保证金扣除
3. 验证：账户余额、持仓、保证金正确

8.2 风控拒绝场景

| 场景 | 触发条件 | 预期结果 |
|------|----------|----------|
| 余额不足 | 可用 < 订单金额 | REJECT: INSUFFICIENT_BALANCE |
| 持仓超限 | 持仓占比 > 95% | REJECT: POSITION_LIMIT_EXCEEDED |
| 保证金不足 | 保证金率 < 100% | REJECT: MARGIN_INSUFFICIENT |
| 价格偏离 | 偏离 > 1% | REJECT: PRICE_DEVIATION_EXCEEDED |

8.3 强制平仓场景

1. 持仓亏损，导致保证金率 < 0.5%
2. 触发强平
3. 验证：持仓清零、保证金扣除、记录强平日志

8.4 指标对比验证

1. 同时接收 Rust 计算的指标和 Python 计算的指标
2. 输出到 indicator_comparison.csv
3. 用户对比差异
