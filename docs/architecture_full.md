四、关键优化点体系
================================================================

┌─────────────────────────────────────────────────────────────────────┐
│ 4.1 架构优化                                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ✅ 移除 panic!/unwrap() 滥用                                        │
│     ─────────────────────────────────────────────                   │
│     - 全部返回 Result<T, Error>                                       │
│     - 使用 thiserror 定义清晰错误枚举                                 │
│     - 错误通过 Trait 接口传播，不使用 panic                          │
│                                                                     │
│  ✅ 优化过量 clone()                                                 │
│     ─────────────────────────────────────────────                   │
│     - 高频路径无 clone                                               │
│     - 策略持仓独立计算（无锁）                                        │
│     - 资金池 RwLock 保护（低频）                                     │
│     - 使用引用 & 和 Arc<Weak> 减少 clone                             │
│                                                                     │
│  ✅ 解决锁瓶颈                                                       │
│     ─────────────────────────────────────────────                   │
│     - Tick接收、指标更新、策略判断全无锁                              │
│     - 锁仅用于下单和资金更新                                          │
│     - 锁外预检所有风控条件                                            │
│     - 使用 parking_lot::RwLock 替代 std::sync::RwLock               │
│                                                                     │
│  ✅ 完善业务错误处理                                                  │
│     ─────────────────────────────────────────────                   │
│     - EngineError / MarketError / AppError 分层                      │
│     - XDataError 业务数据层错误                                      │
│     - 所有错误可追溯、可恢复                                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────┐
│ 4.2 高频路径优化 (O(1) 增量计算)                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  指标计算增量更新:                                                   │
│  ─────────────────────────────────────────────                      │
│  - EMA: 指数移动平均 → 增量计算                                      │
│  - SMA: 简单移动平均 → 增量计算                                      │
│  - RSI: 相对强弱指标 → 增量计算                                       │
│  - MACD: 异同移动平均 → 增量计算                                      │
│  - PineColor: 趋势信号 → MACD+EMA10/20+RSI组合                       │
│                                                                     │
│  K线增量更新:                                                        │
│  ─────────────────────────────────────────────                      │
│  - 当前K线实时增量更新                                               │
│  - 不重算历史K线                                                     │
│  - 三层指标体系: TR(波动率) + Pine颜色 + 价格位置                     │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘

十三、完整架构（异步任务 + 分层自运行）
================================================================

**为什么不用微服务？** 交易系统追求微秒级低延迟，微服务通过网络通信引入数十毫秒延迟。
采用共享内存的数据服务模式，同进程内数据交互零网络开销。

================================================================
十三-A、数据服务模式（低延迟设计）
================================================================

```
┌─────────────────────────────────────────────────────────────────────┐
│  数据服务层（共享内存，同进程）                                     │
├─────────────────────────────────────────────────────────────────────┤
│  DataFeeder (Tick/K线数据服务)                                      │
│  └─ 持续接收数据，按需提供 ws_get_1m(symbol)                        │
├─────────────────────────────────────────────────────────────────────┤
│  IndicatorCache (指标数据服务)                                      │
│  └─ 持续计算指标，按需提供 get(symbol)                              │
└─────────────────────────────────────────────────────────────────────┘
                              ↑ 提供数据
┌─────────────────────────────────────────────────────────────────────┐
│  消费者（引擎层/策略层）                                            │
│  └─ 按需调用 → 共享内存访问 → 微秒级响应                           │
└─────────────────────────────────────────────────────────────────────┘
```

**延迟对比：**
| 架构 | 数据访问延迟 | 适用场景 |
|-----|-------------|---------|
| 微服务 | 10-50ms (网络开销) | 高吞吐、低实时要求 |
| 共享内存 | <1μs (内存访问) | 量化交易、高频交易 |

十三-B、四层自运行架构
================================================================

┌─────────────────────────────────────────────────────────────────────┐
│  数据源层（后台自运行）                                            │
│  StreamTickGenerator → push_tick → DataFeeder                     │
│  作用：持续接收数据，存入内存，供其他组件拉取                       │
└─────────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────┐
│  指标层（后台自运行）                                              │
│  从 DataFeeder 获取 K 线 → 计算指标 → IndicatorCache               │
│  作用：持续计算指标（RSI/EMA/波动率），供其他组件查询              │
└─────────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────┐
│  引擎层（监控波动率触发任务）                                       │
│  监控波动率 → 波动率 > 阈值 → spawn_task                         │
│  职责：只监控波动率，不监控价格                                    │
└─────────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────┐
│  策略层（每个任务独立）                                            │
│  从 DataFeeder 获取价格                                            │
│  从 IndicatorCache 获取指标                                        │
│  策略计算 → 风控 → 下单                                            │
│  平仓完成 → 设置 status=Ended → 自己退出                           │
└─────────────────────────────────────────────────────────────────────┘

### 完整分层架构

```
数据源层（自运行）     ┌─────────────────────────────────────┐
                      │ StreamTickGenerator                 │
                      │   → push_tick → DataFeeder         │
                      └─────────────────────────────────────┘
                                          ↓
指标层（自运行）       ┌─────────────────────────────────────┐
                      │ IndicatorCache                      │
                      │   → 计算 RSI/EMA/波动率              │
                      └─────────────────────────────────────┘
                                          ↓
引擎层                 ┌─────────────────────────────────────┐
                      │ TradeManagerEngine                  │
                      │   → 监控波动率 > 阈值 → spawn_task  │
                      │   → 任务注册表/心跳检查/持久化        │
                      └─────────────────────────────────────┘
                                          ↓
策略层（每个任务）     ┌─────────────────────────────────────┐
                      │ StrategyTask                         │
                      │   → 从 DataFeeder 获取价格           │
                      │   → 从 IndicatorCache 获取指标       │
                      │   → 策略计算 → 风控 → 下单            │
                      └─────────────────────────────────────┘
```

### 引擎层 vs 策略层职责划分

| 触发任务 | ✅ | ❌ |
| 任务注册/移除 | ✅ | ❌ |
| 心跳检查 | ✅ | ❌ |
| 持久化 | ✅ | ❌ |
| 获取价格 | ❌ | ✅ 从 DataFeeder |
| 获取指标 | ❌ | ✅ 从 IndicatorCache |
| 策略计算 | ❌ | ✅ |
| 风控检查 | ❌ | ✅ |
| 下单执行 | ❌ | ✅ |

### TaskState 结构

```rust
pub struct TaskState {
    symbol: String,
    status: RunningStatus,  // Running/Stopped/Ended
    last_beat: i64,
    position_qty: Decimal,
    forbid_until: Option<i64>,
    interval_ms: u64,
}
```

### 异步任务模式

- 不是子线程，是 `tokio::spawn` 的协程
- 引擎 spawn 任务，任务自己循环
- 任务结束自己退出，引擎发现后移除

### 与 Python TradeManager 对应

| Python TradeManager | Rust Engine |
|--------------------|-------------|
| `instances` | `tasks` |
| `start_instance()` | `spawn_task()` |
| `run_once()` | 策略层自己的循环 |
| `monitor()` | `check_heartbeat()` |
| `scan_volatile()` | `check_triggers()` |
