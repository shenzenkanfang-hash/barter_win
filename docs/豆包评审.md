## 二、最高优先级：即时架构整改（0 代码改动，纯规范梳理）
### 1. 强制统一「类型契约」（解决跨 crate 冲突）
**不动代码，只做架构定义**：
- 全局唯一数据源：所有公共类型（`Side/OrderType/TradingAction/RiskCheckResult`）**收敛到 `a_common::models`**
- 禁止行为：任何 crate 禁止重复定义基础交易类型
- 架构规范：`f_engine/interfaces` 只做 **接口(Trait)**，不做公共类型

### 2. 强制收敛「依赖关系」（解决跨层乱依赖）
**最终唯一合法依赖链**：
```
L1 基础设施(a_common) ← 所有下层依赖
L2 数据源(b_data_source) → 只依赖 a_common
L3 信号(c_data_process) → 只依赖 b_data_source + a_common
L4 检查(d_checktable) → 只依赖 c_data_process + a_common
L5 风控(e_risk_monitor) → 只依赖 b_data_source + a_common
L6 引擎(f_engine) → **只依赖 interfaces 接口，不直接依赖任何业务 crate**
```
- 禁止：`f_engine → c_data_process/d_checktable` 直接导入
- 禁止：循环依赖、跨层依赖

### 3. 废弃旧架构「最终确认」
- 全项目唯一合法引擎：`core/engine_v2.rs`
- `engine.rs` 永久废弃，架构文档全部删除旧架构描述

## 三、短期架构标准化（纯设计，后续代码落地用）
### 1. 补齐「接口实现契约」（让 interfaces 层真正生效）
**不动代码，只定义架构绑定关系**：
| Trait 接口 | 必须由哪个 crate 实现 | 架构职责 |
|-----------|---------------------|---------|
| MarketDataProvider | b_data_source | 数据源对接 |
| StrategyExecutor | c_data_process | 信号执行 |
| CheckTableProvider | d_checktable | 检查层执行 |
| RiskChecker | e_risk_monitor | 风控检查 |
| ExchangeGateway | a_common | 网关对接 |

### 2. 落地「依赖反转原则（DIP）」
- 高层（f_engine）**不依赖低层具体实现**，只依赖接口
- 低层（数据源/风控/策略）**负责实现接口**
- 引擎通过 **泛型 + 构造注入** 完成组装（当前 engine_v2 已支持）

### 3. 模块职责「锁死边界」
| 层级 | Crate | 唯一职责 |
|------|-------|---------|
| L1 | a_common | 公共类型/工具/配置/网关 |
| L2 | b_data_source | 喂数据、不做决策 |
| L3 | c_data_process | 算指标、出信号 |
| L4 | d_checktable | 信号校验、价格控制 |
| L5 | e_risk_monitor | 风险拦截、持仓管理 |
| L6 | f_engine | 流程编排、状态管理、订单执行 |
